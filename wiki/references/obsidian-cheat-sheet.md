# Obsidian Cheat Sheet

Complete guide to Markdown formatting and Obsidian wiki-linking conventions.

## Markdown Basics

### Headers
Use `# ` (pound space) for headers:
- `#` - Level 1 header
- `##` - Level 2 header
- `###` - Level 3 header
- `######` - Level 6 header

### Text Formatting

**Bold:** surround text with `**` (double asterisks)
```markdown
This **word** is important. -> This word is important.
```

**Italic:** surround text with `_` (underscores) or `*` (asterisks)
```markdown
_This is_ kind of *important*. -> This is kind of important.
```

### Lists

**Bullet lists:** use `- ` (dash followed by space)
```markdown
- one
- two
- three
```

**Numbered lists:** use numbers followed by period
```markdown
1. First item
2. Second item
3. Third item
```

**Checklists:** use `- [ ]` (dash space [ space ])
```markdown
- [ ] Unchecked item
- [x] Checked item
```

### Tags
Tags are words with `#` in front, with no space between `#` and the word:
```markdown
#word
#tag-name
#multiple-words
```

## Best Practices

### File Naming
- Use lowercase letters
- Separate words with hyphens: `my-note.md`
- Avoid spaces and special characters

### Linking Strategy
- Use descriptive link names
- Link related concepts together
- Create backlinks between related pages
- Use tags for categorization

### Organization
- Group related pages in directories
- Use consistent naming conventions
- Keep index files updated
- Use references/ for external documentation
