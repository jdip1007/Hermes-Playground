        <h1>PubMed Update: FTP Improvements Available for Testing</h1>
 

<a href="https://techbull.nlm.nih.gov/vivisimo/cgi-bin/query-meta?v%3Aproject=technical-bulletin-date&sortby=Date-decending&query=&binning-state=product==PubMedCentral(PMC)"><button class="usa-tag padding-y-05 padding-x-1">Pubmed Central</button></a> <a href="https://techbull.nlm.nih.gov/vivisimo/cgi-bin/query-meta?v%3Aproject=technical-bulletin-date&sortby=Date-decending&query=&binning-state=product==PubMed"><button class="usa-tag padding-y-05 padding-x-1">Pubmed</button></a> 

        <p class="text-base">PubMed Update: FTP Improvements Available for Testing. NLM Tech Bull. 2025 May-Jun;(<a href="/pubs/techbull/mj25/mj25_issue_cover.html">464</a>):e5.</p>


         
        <p class="border-bottom-1px border-primary grid-col-6 padding-bottom-3 margin-bottom-0">June 06, 2025 [posted] 
       


</p>
          
      </div>
    </div>
    <div class="grid-row grid-gap">
 
   
  
<div class="desktop:grid-col-6 tablet:grid-offset-1 padding-y-3 tablet:grid-col-12 mobile:grid-col-12">
 
   

       <p>In response to user feedback, the National Center for Biotechnology Information (NCBI) is announcing an upcoming update to the data provided through <a href="https://pubmed.ncbi.nlm.nih.gov/download/#ftp">PubMed's public FTP servers</a>.</p>
<p>Starting with the annual baseline release in December 2025, the content of PubMed data distributed via FTP will be generated using the same technology as the PubMed website and E-utilities API; this update means the FTP data will match the PubMed website and API.<br /><br /></p>
<h2 class="article-title">Test Site Available Now</h2>
<p>To help users prepare for this transition, a <a href="https://ftp.ncbi.nlm.nih.gov/pubmed/beta-updatefiles/">test FTP site</a> is available now with daily update files generated by the new process. Please note, the test files may differ from the current FTP update files (see Expected Data Differences below).</p>
<p>A complete baseline will not be available on the test site; only update files will be posted.</p>
<p>We encourage users to review the sample files, test their workflows, and provide feedback via the <a href="https://support.nlm.nih.gov/support/create-case/">National Library of Medicine (NLM) Help Desk</a>.</p>
<p>Access the <a href="https://ftp.ncbi.nlm.nih.gov/pubmed/beta-updatefiles/">Test FTP Site</a>. <br /><br /></p>
<h2 class="article-title">Expected Data Differences</h2>
<p>FTP users will notice some differences in data formatting and content. Most of these changes improve standardization and data richness without affecting the underlying meaning of the data. These differences include:</p>
<p><strong>Availability of Additional Data </strong>- PubMed data may be augmented with additional data from PubMed Central (PMC) when available. This additional data was already integrated on the PubMed website and API and will now be included in the FTP files. This change will ensure that all users have access to consistent, up-to-date PubMed data.</p>
<p>Examples include:</p>
<ul>
<li>More robust reference data supplied by PMC.</li>
<li>Inclusion of Conflict of Interest (COI) statements and additional &lt;OtherAbstract&gt; content from PMC.</li>
<li>Inclusion of PMC Release Date even after the release date has passed.</li>
<li>&lt;Pagination&gt; will include &lt;StartPage&gt; and &lt;EndPage&gt; in addition to &lt;MedlinePgn&gt; when available from PMC.</li>
</ul>
<p>Due to this augmented data, daily update files on the test site may contain additional PMIDs that are not included in the current FTP files since the current files are not updated with the PMC data.</p>
<p><strong>Formatting, Normalization, and Ordering</strong> &ndash; Some data may be formatted or ordered differently from the current FTP files. The structure of files and meaning of the data are not changed. Examples include:</p>
<ul>
<li>Differing order of attributes or child elements.</li>
<li>Differences in capitalization of labels and some text.</li>
<li>Single digit dates will not be preceded by a zero (e.g., "1" instead of "01").</li>
<li>Author identifiers will not be preceded by the URL. The ID source will remain as an attribute in &lt;Identifier&gt;.</li>
</ul>
<p><strong>Schedule of Updates &ndash;</strong> Please note that daily update files for the current FTP and the test FTP site are generated using different processes on separate schedules. This means updated PMIDs may appear on different days between the two sites.<br /><br /></p>
<h2 class="article-title">Share Your Feedback</h2>
<p>Please&nbsp;<a href="https://support.nlm.nih.gov/support/create-case/">reach out to us</a>&nbsp;if you have questions or would like to provide feedback.</p>
<p>For more information about accessing PubMed data via FTP, please see: <a href="https://pubmed.ncbi.nlm.nih.gov/download/">Download PubMed Data</a>.</p>
<p>NLM will share additional details as the transition date approaches. Stay tuned to the <a href="https://www.nlm.nih.gov/pubs/techbull/">NLM Technical Bulletin</a> for updates.</p>
