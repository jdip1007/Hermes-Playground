# Hermes Agent 更新日志（2026-04-11 ~ 2026-04-17）

641 个 commit（366 + 275），发布 **v0.10.0 (2026.4.16)**。以下为重点内容。

## 新增 Provider

| Provider | 说明 |
|----------|------|
| **AWS Bedrock** | 原生 Converse API（`agent/bedrock_adapter.py`，1098 行）。双路径：Claude 模型走 AnthropicBedrock SDK（保留 prompt caching/thinking），非 Claude 走 boto3 Converse API（Nova/DeepSeek/Llama/Mistral）。IAM + API Key 两种认证，动态模型发现，7 个模型定价 |
| **Google Gemini CLI OAuth** | 通过 Cloud Code Assist 后端接入 Gemini（`agent/google_oauth.py`，1048 行 + `agent/gemini_cloudcode_adapter.py`）。PKCE Authorization Code flow，跨进程文件锁，支持免费层（个人账户）和付费层（GCP project） |
| **Ollama Cloud** | 内置 provider（与 gemini/xai 平级）。`OLLAMA_API_KEY` 认证，models.dev 集成，动态模型发现 + 磁盘缓存（1h TTL），保留 `model:tag` 格式 |

## Nous Tool Gateway

把 web 搜索、TTS、浏览器、图片生成的 API 调用路由到 Nous 托管的统一网关，用户无需自备各家 API key。从隐藏环境变量改为订阅制自动检测：

```yaml
# config.yaml — 按工具类别 opt-in
web:
  use_gateway: true
tts:
  use_gateway: true
```

`hermes model` 登录后展示可用工具列表，用户选择启用全部 / 仅未配置的 / 跳过。

## 上下文压缩 v3

### 三阶段预处理（无 LLM 调用，零 token 成本）

| Pass | 操作 | 效果 |
|------|------|------|
| Pass 1 | MD5 去重 | 同一文件读 5 次只留最新一份，旧副本标记 `[Duplicate tool output]` |
| Pass 2 | Smart Collapse | 旧工具输出替换为信息化单行摘要（如 `[terminal] ran npm test → exit 0, 47 lines`） |
| Pass 3 | tool_call 参数截断 | >500 字符截到 200（修复 `write_file` 50KB 参数残留） |

### 摘要模板升级（v2 → v3）

- `Progress > Done`（自由文本）→ `Completed Actions`（强制编号 `N. ACTION target — outcome [tool: name]`）
- 新增 `Active State` 段（工作目录/分支/测试状态/运行中进程）
- 删掉 `Tools & Patterns` 段（信息融入 Completed Actions 的 `[tool: name]`）
- 摘要预算从 `budget × 2` 降到 `budget × 1.3`

### 防抖 + 尾部保护

- **Anti-Thrashing**：连续 2 次压缩节省 <10% 则跳过
- **Failure cooldown**：RuntimeError 600s / 瞬态 60s（区分配置错误 vs 临时故障）
- **v0.10.0 修复**：新增 `_ensure_last_user_message_in_tail()`，最后一条用户消息永不被压进摘要区域（#10896）

## 插件系统增强

### 命名空间技能（Plugin Namespaced Skills）

插件可以注册带命名空间的技能（`plugin:skill`），避免与内置技能重名：

```python
ctx.register_skill(name="deploy", path=..., description="...")
# 实际存为 "myops:deploy"，agent 调用 skill_view("myops:deploy")
```

- 不进系统提示索引（零 token 开销）
- bundle context banner 列出同一插件的兄弟技能
- 完整防护：disable 检查、平台兼容、注入模式扫描

### pre_tool_call 阻止工具执行

`pre_tool_call` 钩子从通知型升级为可阻断：

```python
def my_hook(tool_name, args):
    if "rm -rf" in args.get("command", ""):
        return {"action": "block", "message": "Blocked by policy"}
```

第一个 `{"action": "block"}` 生效，工具跳过执行，message 作为 tool result 返回给模型。

### register_command()（v0.10.0）

插件可以注册自定义斜杠命令：

```python
ctx.register_command(name="deploy", description="...", handler=my_handler)
```

自动出现在 Telegram bot 菜单和 CLI 自动补全中。与内置命令冲突检测。

### dispatch_tool()（v0.10.0）

插件斜杠命令处理器可以通过注册表调度工具调用，自动注入 parent agent 上下文：

```python
result = ctx.dispatch_tool("delegate_task", {"task": "..."})
```

### Dashboard 插件系统 + 主题系统（v0.10.0）

插件可以向 Web Dashboard 添加自定义标签页（`plugins/<name>/dashboard/manifest.json` + `dist/index.js`）。同时新增 Dashboard 主题系统，支持实时切换。

## Gateway 代理模式（Proxy Mode）

网关和 Agent 可以分开部署。典型场景：网关跑在 Docker 处理 Matrix E2EE 加密，Agent 跑在 macOS 主机访问本地文件和记忆。

```yaml
gateway:
  proxy_url: "http://host.docker.internal:8080"
```

通过 HTTP + SSE streaming 转发，`X-Hermes-Session-Id` header 保证跨请求 session 连续。
