# Hermes Agent Changelog (2026-04-09 ~ 2026-04-11)

293 commits, 35+ new features. Highlights below.

---

## Architectural Changes

### Pluggable Context Engine

**Core Change**: Context management refactored from hardcoded `ContextCompressor` to a pluggable `ContextEngine` ABC (`agent/context_engine.py`).

```yaml
# config.yaml — Switch context engine
context:
engine: "compressor"   # Default; set to a plugin name to switch (e.g., "lcm")
```

The ABC requires implementing 3 core methods:
- `name` — Engine identifier
- `should_compress(prompt_tokens)` — Whether compression is needed
- `compress(messages, current_tokens)` — Executes compression, returns new message list

Optional methods: `on_session_start/end`, `get_tool_schemas` (engine can expose tools to agent), `update_model` (updates on model switch).

The built-in `ContextCompressor` now inherits from `ContextEngine` and serves as the default implementation. Third-party plugins reside in `plugins/context_engine/<name>/`.

### Unified File Synchronization (`file_sync.py`)

New module `tools/environments/file_sync.py`: Provides unified file synchronization for SSH/Modal/Daytona backends, based on mtime+size change tracking, with deletion detection and transaction rollback support. Docker/Singularity use bind mounts and do not require this.

### Background Process Monitoring (`watch_patterns`)

The `terminal` tool added a `watch_patterns` parameter to notify the agent in real-time when background process output matches specified strings:

```python
terminal(command="pytest -v", background=True, watch_patterns=["ERROR", "FAIL", "listening on port"])
```

Built-in rate limiting (max 8 notifications per 10-second window), auto-disables after 45 seconds of sustained overload to prevent notification storms.

| Feature | Description |

| **require_mention** | Requires @mention in channels to respond (default true) |
