# v0.12.0 (v2026.4.30) — The Curator Release

**1,096 commits · 550 merged PRs · 1,270 files changed · 217,776 insertions**

> Autonomous Maintenance Release — Hermes begins **maintaining itself**. The background Curator automatically scores / merges / prunes the skill library on an independent cron ticker; self-improvement loop significantly upgraded; 4 new inference providers; 18-19 platforms (Yuanbao + Teams); native Spotify tools, Google Meet plugin; ComfyUI and TouchDesigner-MCP moved from optional → bundled; TUI cold start reduced by ~57%.

## ✨ Top New Features

### 1. Autonomous Curator — Self-Maintaining Skill Agent

**`hermes curator` upgraded to a background agent**, running automatically every 7 days on the gateway's cron ticker.
- **Per-run reports**: `logs/curator/run.json` + `REPORT.md`
- **`hermes curator status`**: Summary of the latest run + ranking of most/least used skills
- **Unified under the `auxiliary.curator` slot**: Select the curator model directly in `hermes model`, configured via dashboard
- **Archive classification**: consolidated (merged into another skill) vs. pruned (independently removed), jointly determined by model + heuristics
- Strictly protects bundled / hub-installed skills; pinned skills prevent writes from `skill_manage`

Source: `agent/curator.py` (1781 lines), `agent/curator_backup.py` (693 lines), `hermes_cli/curator.py` (598 lines).

### 2. Self-improvement Loop — Comprehensive Upgrade

Per-turn background review fork (the core of Hermes' self-improvement):
- **Rubric-based grading** (replaces previous free-text "should it be updated?")
- **Active-update bias**: Favors updating skills that were recently loaded
- **`references/` / `templates/` sub-file awareness**
- **runtime inheritance**: fork actually inherits the parent agent's provider/model/credentials
- **scoped toolsets**: fork restricts memory + skills (cannot use shell / web)
- **clean shutdown** + **clean context**
