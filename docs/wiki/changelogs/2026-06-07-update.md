## 1. Compression — 摘要时间锚定

**Commit**：`d87f2939 feat(compression): temporal anchoring in compaction summaries (#41102)`，作者 teknium1，2 文件 **+146**：

- `agent/context_compressor.py` **+32/-1**：`_generate_summary()` 在两条 summarization prompt 路径（首次压缩 + 迭代更新）都注入新的 `_temporal_anchoring_rule` 字符串，规则要求把"打算 X"改写为"已于 YYYY-MM-DD X 完成"（commit 页给出示例：`"email John about the proposal" → "Sent the proposal email to John on 2026-06-07"`），日期经 `hermes_time.now()` 取 `YYYY-MM-DD` 格式。
- 时钟异常时**降级（fail-open）**：日期解析失败则**省略**该规则而非阻塞 compaction。
- `tests/agent/test_context_compressor_temporal_anchoring.py` **新建 +114**：验证规则出现在 prompt、日期解析正确、时钟错时压缩仍能继续。

**意义**：续接会话时，模型不再把"已完成动作"误读为"待办"再次发起 — 这是长会话被压缩后行为漂移的常见根因。

## 2. Compression — gpt-5.5 on Codex OAuth 的 85% 压缩触发阈值

**Commit**：`0524c9bd feat(compression): raise compaction trigger to 85% for gpt-5.5 on Codex OAuth`，5 文件 **+218/-4**。

<a id="4-compression-sentinel"></a>
## 4. Compression — 不再覆盖 `-1` post-compression 哨兵

**Commit**：`3c8f1ded fix(compression): don't overwrite the -1 post-compression sentinel`，作者 teknium1 + davidgut1982。

post-compression 哨兵值 `-1` 在 token 计费路径上承担"该回合刚刚被压缩、计费应跳过"的语义；早先某条更新路径会用真实 token 数把 `-1` 覆盖回去，导致已压缩回合被重复计费。本修复在覆盖前显式检查 `-1` 哨兵并保留。
