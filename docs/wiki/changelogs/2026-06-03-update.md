### 1.2 关键设计：hot path 零成本

`hermes_cli/plugins.py:1574 has_hook(hook_name)` 让 emit 站点在"无订阅者"时短路：

- payload dict 构建不发生（**只在有 listener 时才构建**）
- request payload 在 middleware 不重写时**按引用返回**
- response payload sanitization 不再嵌入原始 response 对象

### 1.3 NeMo-Relay 可选插件

`plugins/observability/nemo_relay/` 三文件（**568 + 368 + 20 行**）作为新 hook 第一个 in-repo 消费者，与既有 `plugins/observability/langfuse` 并列：

- `__init__.py 568 行` — 主插件实现

`pyproject.toml` 新加可选依赖；`docs/observability/README.md 316 行`总览所有 observability hook 契约；`hermes_cli/plugins.py +16` 行新增 hook 名注册；`website/docs/user-guide/features/hooks.md +4` 行用户指南补充。

## 5. Desktop session/scroll/sticky/clamp wave

**~12 commits**，集中在 thread-virtualizer / chat 滚动定位 / sticky human bubble 的多场景纠错。

### 5.1 In-flight new chats 不再消失或匿名（3 commits）

- `55a76ec66 fix(desktop): keep in-flight new chats from vanishing on refresh` —— 新 session 的首条 user message 直到 turn 持久化才 flush 到 SessionDB，所以 mid-response 时 `message_count=0`；`refreshSessions()` 用 `min_messages=1` list，把还没 flush 的 in-flight chat 排除。修。
- `e0a999aa8 fix(desktop): label in-flight new chats with the first message` —— 之前优化乐观 sidebar 行用 `null` preview，多个 "Untitled session" 并存。Seed 用 user 首条消息作为预览。
- `28f1590b7 fix(desktop): stop background session messages bleeding into the active transcript` —— 后台 still-busy session 仍在 emit `updateSessionState()`；每次都无条件调 `syncSessionStateToView()`。`flushPendingViewState()` 保护错 session 写入 view，但**每帧只有一个 rAF**且 `pendingViewStateRef` 只持最新 writer，背景写就盖掉前景写。修：staging 改成按 sessionId keyed 的 map，只 flush 当前 session 的写。

### 5.2 Scroll anchoring 三连

- `e67ab2e04 fix(desktop): stop chat scroll jumping by disabling native scroll anchoring` —— `@tanstack/react-virtual` 自己调 scrollTop 修测量差，浏览器默认 `overflow-anchor: auto` 同时修同一 delta，**双修**导致 lurch（最严重在 Windows 鼠标滚轮粗 notch）。
- `3aa24e261 fix(desktop): stop chat scroll backward-jump from content-growth interim scrolls` —— `thread-virtualizer.tsx` 之前在 scrollTop 减少 >1px 时 disarm sticky-bottom，但**内容增长**时浏览器会 interim emit scrollTop 减小（虚拟化测量 / 流式 token / Streamdown-Shiki 重新 tokenize），rAF-scheduled `pinToBottom` 还没跑，`programmaticScrollPendingRef = 0`——误判为用户滚动。
- `c930a49ce fix(desktop): honor upward wheel scroll in long threads` —— 长 thread 中向上 wheel 之前被吞，1 行新增 + 34 测试。
- `0caa23788 fix(desktop): prevent IME Enter from splitting messages and viewport resize from disarming scroll anchor`：
  - 韩/日/中 IME composing 时 Enter 完成 preedit 触发 `submitDraft()` —— `handleEditorKeyDown` 没 check `event.nativeEvent.isComposing`。修：`apps/desktop/src/app/chat/composer/index.tsx:581 if (event.nativeEvent.isComposing) return`。
  - composer expand/collapse 缩 viewport 时浏览器调 scrollTop，onScroll 误读为用户滚 up。修：`apps/desktop/src/components/assistant-ui/thread-virtualizer.tsx:230 lastClientHeightRef`，disarm 条件需同时 stable scrollHeight + stable clientHeight。

- `93228d529 fix(desktop): persist pins, reconnect after sleep, dedupe session search` —— 4 修：
  1. **Pin 丢失直到 refresh** —— 分页 in-memory session list join 后 pinned chat 不在最近页就被驱逐；`mergeWorkingSessions → mergeSessionPage` 同时保留 pinned rows（按 live id 或 lineage root 匹配）。

## 10. Dashboard auth — WS origin + PTY internal credential

### 10.1 OAuth 闸门后非 Web WS origin 信任

**Commit**：`dd28f2ac9 fix(dashboard): trust non-web WS origins on OAuth-gated binds after ticket auth (#37870)`：

把 #37747 推广。`_ws_host_origin_is_allowed` WS Origin guard 之前只在**非 OAuth-gated** 绑定时信任打包 Electron app 非 web origin（`file://` / `null` / `app://`）。Hermes Desktop renderer 走 `file://` 加载，所以驱动远端 OAuth-gated gateway 时 `/api/ws` upgrade 被 403 拒绝，**虽然 `_ws_auth_ok` 上一行刚验过单次 `?ticket=`**。

这个 guard 只在 `_ws_auth_ok` 接受 WS 凭据**之后**跑（每种模式真正的 auth 边界都在前面）：

- loopback bind → 历史 dashboard session token
- 非 loopback `--insecure` → 历史 session token（Tailscale / LAN，#37747）
- OAuth-gated public bind → 单次性、30s-TTL、身份绑定 `?ticket=`

非 web origin 只能从原生 client 来；DNS-rebinding 攻击总走 http(s) origin 且仍按绑定 host match 检。所以一旦上游凭据检过了，Origin guard 对非 web origin 无加值。loopback / 非 gated 特例都 collapse 到 `return True`。

http(s) origin 保持严格同 host 检查，浏览器 DNS-rebinding 仍被挡。

### 10.2 PTY child WS 进程内 internal credential

**Commit**：`fd1ec8033 fix(dashboard): authenticate server-spawned PTY child WS with a process-internal credential`：

嵌入式 TUI PTY child 接两个 server-internal WebSocket：`/api/ws`（主 JSON-RPC gateway 后端）和 `/api/pub`（事件 sidecar）。两 URL 在 `web_server.py` 服务器端构建并通过 child env 传入。

OAuth-gated 模式下 `_ws_auth_ok` 无条件拒 `?token=<_SESSION_TOKEN>`——泄漏 session token 不应该在闸门启用后还能拿 WS。但 `_build_gateway_ws_url()` 还在只发 `?token=`，没有 gated 模式分支（其兄弟 `_build_sidecar_url` 已加 ticket 分支；gateway url builder 漏了）。所以 TUI child 的 `/api/ws` upgrade 4401 → "gateway websocket connection failed" → "gateway startup timeout"，**每个 gated 部署的嵌入式 chat 都不能用**。

修：mint-once 进程内 credential，PTY child 用它鉴权。`e114b31ed test(dashboard): direct unit coverage for internal WS credential + docstring fix` 直接覆盖 `TestInternalCredential` 块（mint-once 稳定、多次使用、未 mint 拒绝、空值、错值、reset-and-remint、ticket-store 独立）。

### 10.3 PTY 维度 WSL2 钳制

**Commit**：`966630563 fix(dashboard): clamp PTY resize dimensions for WSL2 winsize garbage (#38200)`：

WSL2 winsize 探测坏掉时报 `columns=131072, rows=1`。dashboard `/chat` tab 转发 xterm.js 尺寸到 `PtyBridge.resize()`，用 `struct.pack` 打 unsigned short → 131072 > 65535 抛 `struct.error`（之前**只** catch `OSError`） → resize 路径挂，TUI 留在 1 行宽得离谱屏布局，看起来文字消失。

## 16. TUI — 持久化 trail OOM + `/save` Hermes home + 右键复制

### 16.1 Trail 持久化 OOM 修复

**Commit**：`e76d8bf5a fix(tui): stop persisting full tool output in trail lines (silent OOM death)`：

heavy `--tui` session（browser snapshots、大 tool output）几分钟内**沉默 OOM-kill** Node 父进程——关闭 gateway child 的 stdin，用户只看到 "gateway exited" / stdin EOF。CLI 免疫。根因：每个完成的 tool verbose trail 行嵌入 ≤16KB `result_text`，持久化在 transcript `Msg.tools[]` 整 session，默认渲染 EXPANDED——一次为 800 messages 全部建 Ink render-node tree。几百 MB 内存，**远低于** 2.5GB memory-monitor 退出阈值，死亡完全没归因。

- `ui-tui/src/lib/text.ts` —— 持久化 verbose tool-trail 块改为 `VERBOSE_TRAIL_MAX_CHARS=800` / 12 行 preview，非 16KB live-render budget。保留 trail 字符串掉 ~17×（12.2MB → 0.7MB @ 800 msgs）；live streaming tail 仍用更大 LIVE_RENDER budget。
