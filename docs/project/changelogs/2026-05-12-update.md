# Post-v0.13.0 Sync — 2026-05-12

Incremental updates roughly one week after the v0.13.0 release. Major changes prior to the next official release.

## Major Features

### 1. Cross-session 1h prefix cache for Claude (PR #23828)

See the full article on [Prompt Caching Optimization](../concepts/prompt-caching-optimization.md). Added `apply_anthropic_cache_control_long_lived` and `mark_tools_for_long_lived_cache` to `agent/prompt_caching.py`, specifically designing a 4-breakpoint layout for the three paths: Claude on Anthropic / OpenRouter / Nous Portal:

| Breakpoint | Location | TTL |
|------|------|-----|
| tools[-1] | End of tools array | **1h** |
| system block[0] | Stable prefix block in system message | **1h** |
| 2nd/1st last non-system messages | Rolling window | 5m |

The first session writes to the 1h cache, and the second session hits it immediately on **cold start**. Prerequisite: System messages must be split by `prompt_builder` into ordered content blocks, with block[0] having stable bytes.

Follow-up fix in PR #24151: Routes Nous Portal **Qwen** to the Portal-Claude cache pathway as well, preventing missed 1h caches.

### 2. `feat(agent): per-turn file-mutation verifier footer` (PR #24498)

Added `_file_mutation_verifier_enabled()` and `_format_file_mutation_failure_footer()` to `run_agent.py:5453-5478`. At the end of each turn, if `write_file` or `patch` fails (due to syntax/path/permission issues), a footer is automatically appended to the assistant's reply listing up to 10 failed paths plus the first error.

- Config: `display.file_mutation_verifier` (bool, default true)
- Env override: `HERMES_FILE_MUTATION_VERIFIER=0`

Complements the post-write delta lint from v0.13: the lint catches errors at the tool layer, while the verifier exposes them to the user at the reply layer.

### 3. `feat(security): supply-chain advisory checker + lazy-install framework + tiered install fallback` (PR #24220)

New files: `hermes_cli/security_advisories.py`, `tools/lazy_deps.py`.

- Scans for known supply-chain advisories at startup (CLI, Gateway) (e.g., May 2026 Mini Shai-Hulud worm), providing prominent warnings on matches.
- The `security.allow_lazy_installs` config (default true) controls whether "on-demand lazy installs" of dependencies are allowed.
- Tiered install fallback: Gracefully degrades in tiers when dependency installation fails, instead of crashing with a traceback.

### 4. `feat: expose HERMES_SESSION_ID to agent tools via ContextVar + env` (PR #23847)

Added `_SESSION_ID` ContextVar to `gateway/session_context.py:58`, paired with setting `os.environ["HERMES_SESSION_ID"]` in `run_agent.py:1913`, allowing tool subprocesses (shell/skill scripts/mcp_serve) to access the current session ID.

Also supports using the `${HERMES_SESSION_ID}` placeholder in SKILL.md (`hermes_cli/config.py:1146`).

### 5. `Use Nous Portal as model metadata authority` (PR #24502, #24509, e2b713c)

- Model picker/metadata resolution treats **Nous Portal** as the authoritative catalog (replacing OpenRouter fallback).
- Known providers skip OpenRouter probing (PR e2b713c).
- Paid recommendations take the union of Portal and static lists (PR #24509).
- Portal-flagged free models are displayed even if the local curated list is outdated (PR #24082).

### 6. `feat(computer-use): refresh cua-driver on hermes update + add install --upgrade` (PR #24063)

The cua-driver (computer use) now follows `hermes update`, and provides an explicit upgrade path via `hermes install --upgrade`.

### 7. `feat(ui-tui): resolve markdown links to readable page titles` (PR #24013)

When displaying markdown links in the TUI, it resolves the remote `<title>`, rendering `https://example.com/foo` as the actual page title.

### 8. `add client marker tag on aux inference requests` (94d9db7)

Adds a client marker tag to auxiliary model requests, facilitating request attribution for Nous Portal/upstream.
