## Nous Tool Gateway

Routes API calls for web search, TTS, browser, and image generation to a unified gateway hosted by Nous, so users no longer need to manage individual API keys. Switched from hidden environment variables to subscription-based auto-detection:

```yaml
# config.yaml — opt-in by tool category
web:
  use_gateway: true
tts:
  use_gateway: true
```

After `hermes model` login, displays available tools list. Users can choose to enable all / only unconfigured ones / skip.

## Context Compression v3

### Three-Stage Preprocessing (No LLM calls, zero token cost)

| Pass | Operation | Effect |
|------|------|------|
| Pass 1 | MD5 Deduplication | Reading the same file 5 times keeps only the latest copy; old copies marked `[Duplicate tool output]` |
| Pass 2 | Smart Collapse | Old tool outputs replaced with informative single-line summaries (e.g., `[terminal] ran npm test → exit 0, 47 lines`) |
| Pass 3 | `tool_call` Parameter Truncation | Truncates >500 chars to 200 (fixes `write_file` 50KB parameter residue) |

### Summary Template Upgrade (v2 → v3)

- `Progress > Done` (free text) → `Completed Actions` (mandatory numbering `N. ACTION target — outcome [tool: name]`)
- Added `Active State` section (working directory/branch/test status/running processes)
- Removed `Tools & Patterns` section (info merged into `[tool: name]` in Completed Actions)
- Summary budget reduced from `budget × 2` to `budget × 1.3`

### Debounce + Tail Protection

- **Anti-Thrashing**: Skips compression if two consecutive compressions save <10%
- **Failure cooldown**: RuntimeError 600s / Transient 60s (distinguishes config errors from temporary failures)
- **v0.10.0 Fix**: Added `_ensure_last_user_message_in_tail()`, the last user message is never compressed into the summary area (#10896)

## Plugin System Enhancements

### Namespaced Skills (Plugin Namespaced Skills)

Plugins can register namespaced skills (`plugin:skill`) to avoid naming conflicts with built-in skills:

```python
ctx.register_skill(name="deploy", path=..., description="...")
# Actually stored as "myops:deploy", agent calls skill_view("myops:deploy")
```

- Not indexed in system prompt (zero token overhead)
- Bundle context banner lists sibling skills from the same plugin
- Full safeguards: disable checks, platform compatibility, injection mode scanning

### `pre_tool_call` Blocks Tool Execution

`pre_tool_call` hook upgraded from notification-only to blocking capability:

```python
def my_hook(tool_name, args):
    if "rm -rf" in args.get("command", ""):
        return {"action": "block", "message": "Blocked by policy"}
```

The first `{"action": "block"}` takes effect, tool execution is skipped, and the message is returned to the model as a tool result.

### `register_command()` (v0.10.0)

Plugins can register custom slash commands:

```python
ctx.register_command(name="deploy", description="...", handler=my_handler)
```

Automatically appears in Telegram bot menu and CLI auto-completion. Includes conflict detection with built-in commands.

### `dispatch_tool()` (v0.10.0)

Plugin slash command handlers can dispatch tool calls via the registry, automatically injecting parent agent context:

```python
result = ctx.dispatch_tool("delegate_task", {"task": "..."})
```

### Dashboard Plugin System + Theme System (v0.10.0)

Plugins can add custom tabs to the Web Dashboard (`plugins/<name>/dashboard/manifest.json` + `dist/index.js`). A new Dashboard theme system is also added, supporting real-time switching.

## Tool Auto-Discovery

`tools/registry.py` adds `discover_builtin_tools()`: AST scans `tools/*.py`, automatically imports any file with a top-level `registry.register()` call. Adding new tools no longer requires manually editing the import list in `model_tools.py`.

## Gateway Proxy Mode

The gateway and agent can be deployed separately. Typical scenario: Gateway runs in Docker handling Matrix E2EE encryption, Agent runs on macOS host accessing local files and memory.

```yaml
gateway:
  proxy_url: "http://host.docker.internal:8080"
```

Forwards via HTTP + SSE streaming, `X-Hermes-Session-Id` header ensures session continuity across requests.
