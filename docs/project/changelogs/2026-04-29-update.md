# Hermes Agent 更新日志（2026-04-19 ~ 2026-04-29）

发布 **v2026.4.23**，182 个 commit。核心主题：**平台适配器插件化**（架构级）+ **Curator 后台技能维护**（新模块）+ **MiniMax OAuth / Vercel Sandbox / 元宝 / IRC** 等新接入。

---

## 架构级变更

### 平台适配器插件化（重大架构）

`gateway/platform_registry.py` 引入 `PlatformRegistry` 单例 + `PlatformEntry` dataclass，让任何人都可以把新平台（IRC、Viber、Line 等）以**纯插件**形式接入，无需改 gateway 核心代码。

**12 个集成点全覆盖**：webhook 投递、PLATFORM_HINTS、`get_connected_platforms`、cron 投递、动态 toolset 生成、setup wizard、统一的设置流程等。bundled 插件平台（如 IRC）启动时自动加载。

新 PluginContext API：

```python
def register(ctx):
    ctx.register_platform(
        name="irc",
        label="IRC",
        adapter_factory=create_irc_adapter,
        check_fn=check_irc_available,
        validate_config=validate_irc_config,
        required_env=["IRC_NICK", "IRC_PASS"],
        install_hint="pip install ...",
    )
```

**IRC 参考实现**（`plugins/platforms/irc/`）：纯 stdlib asyncio，零外部依赖，TLS、PING/PONG、nick collision、NickServ 自动鉴权、频道寻址（`nick: msg`）、Markdown 剥离、消息分片、交互式 setup 向导。

详见 [Messaging Gateway Architecture](concepts/messaging-gateway-architecture.md)。

## 新模块：Curator — 后台技能维护

`agent/curator.py`（869 行）+ `hermes_cli/curator.py`（235 行）+ `tools/skill_usage.py` —— **辅助模型驱动**的后台维护机制，定期审查 agent 创建的 skill，跟踪使用情况，按状态机转换归档闲置 skill。

**不变量**：
- 永不触碰 bundled / hub-installed skill（双过滤）
- 永不自动删除——只归档，可 `hermes curator restore <skill>` 恢复
- Pinned skills 跳过所有自动转换
- 用 aux client，永不污染主 session 的 prompt cache

**触发**：inactivity-triggered（无 cron 守护进程）。CLI/Gateway 启动时检查，满足 `interval_hours`（默认 168h = 7 天）+ `min_idle_hours`（默认 2h）才跑。Gateway 也 hook 进 cron-ticker 线程定期检查。

**状态机**：`active → stale → archived`，重新使用回到 active。纯函数式，无 LLM 调用。仅在需要整合重叠 + 修补漂移时才 fork AIAgent 介入。

**CLI**：`hermes curator {status,run,pause,resume,pin,unpin,restore}`，对应 `/curator` 斜杠命令。

详见 [Skills System Architecture](concepts/skills-system-architecture.md)。

## Memory 系统

### `on_session_switch` provider 钩子

之前 provider 只在初始化时收到一次 `session_id`。但 `session_id` 会在 `/resume`、`/branch`、`/reset`、`/new`、上下文压缩等场景被**重新分配**——provider 不知道，后续写入会落到错误的 session 记录里。

`agent/memory_manager.py:on_session_switch()` 现在会在 session_id 改变时调用所有 provider 的 `on_session_switch(new_session_id, parent_session_id, reset, **kwargs)`，让 provider 刷新缓存的 per-session 状态。

详见 [Memory System Architecture](concepts/memory-system-architecture.md)。
