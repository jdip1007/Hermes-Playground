# Hermes Agent 更新日志（2026-04-17 ~ 2026-04-18）

v0.10.0 发布后 410 个 post-release commit，56 个 feat。核心主题是**架构级重构**（Transport ABC）、**扩展能力**（Shell Hooks、新 Provider）和**可观测性**（subagent 追踪、账户限制）。

---

## 架构级变更

### Provider Transport ABC（重大架构重构）

新增 `agent/transports/` 目录（1217 行，7 个模块），用统一的 `ProviderTransport` ABC 抽象了所有 provider 的 API 数据路径：

```
agent/transports/
├── base.py              # ProviderTransport ABC（89 行）
├── types.py             # NormalizedResponse（142 行）
├── __init__.py          # 注册表 + get_transport() 惰性发现（51 行）
├── anthropic.py         # AnthropicTransport（177 行）
├── chat_completions.py  # ChatCompletionsTransport（387 行）
├── codex.py             # ResponsesApiTransport（217 行）
└── bedrock.py           # BedrockTransport（154 行）
```

**核心抽象**：`convert_messages → convert_tools → build_kwargs → normalize_response`，外加三个可选钩子（`validate_response`、`extract_cache_stats`、`map_finish_reason`）。

替代了 `run_agent.py` 中 10+ 处散落的 `if api_mode == "anthropic_messages": ... elif ...` 分支判断。辅助客户端也迁移到 Transport（压缩、记忆 flush、session summary 路径）。

详见 [Provider Transport Architecture](concepts/provider-transport-architecture.md)。

### Shell Hooks

Hook 回调不再局限于 Python。用户可以在 `config.yaml` 声明 shell 脚本：

```yaml
hooks:
  pre_tool_call:
    - command: /path/to/my-hook.sh
  subagent_stop:
    - command: /path/to/audit.sh
```

脚本从 stdin 收 JSON 事件，从 stdout 返回 JSON 决策。关键设计：`subprocess.run(shell=False) + shlex.split` 防 shell injection；首次使用按 `(event, command)` 对询问用户同意并存 allowlist；`hermes hooks list/test/revoke/doctor` CLI 子命令；Claude Code 兼容响应格式。新增 `subagent_stop` 事件。

### Delegate Orchestrator + 跨 Agent 文件协调

`delegate_task` 新增 `role='leaf'|'orchestrator'`，配合 `delegation.max_spawn_depth`（默认 1 扁平，2-3 解锁嵌套）启用多级委派。

并发子 agent 现在共享文件状态视图，父 agent 汇总时能看到所有子 agent 的文件操作，防止并发 patch 丢失。

## 可观测性

### Subagent Spawn 观测性 Overlay（TUI）

针对递归 `delegate_task` 扇出的**实时 + 事后审计**视图，Ink overlay 支持任意深度+宽度的嵌套子 agent 树。每个 subagent 事件现在携带：
- `subagent_id`、`parent_id`、`depth`、`model`、`tool_count`
- `subagent.complete` 额外携带 input/output/reasoning tokens、cost、api_calls、files_read/files_written

### /usage 账户限制

`/usage` 命令在 token 表格下追加 **provider 侧账户配额/额度**信息。CLI 走 `concurrent.futures.ThreadPoolExecutor(max_workers=1)` + 10s 超时，gateway 走 `asyncio.to_thread`，慢 provider 不影响交互。新模块 `agent/account_usage.py`（326 行，`fetch_account_usage` + `render_account_usage_lines`）提供 per-provider 抽象。

### Dashboard 增强

- 跟踪每个 session 真实的 API 调用次数
- 新增"更新 hermes"和"重启 gateway"按钮
- 主题系统 + 插件系统继续完善

## 平台增强

### 插件斜杠命令跨平台原生化

`register_command()` 注册的插件命令现在在每个 gateway 平台都原生展示：Discord native slash 选择器 / Telegram BotCommand 菜单 / Slack `/hermes` 子命令。新增 `args_hint` 参数支持参数选择器。

**决策型 command hook**：`command:<name>` 钩子可返回 `{"decision": "deny|handled|rewrite|allow"}` 在核心处理前拦截。向后兼容 fire-and-forget 遥测钩子。

### 企业微信 WeCom

QR 扫码认证 + 交互式设置向导（`hermes_cli/gateway.py:_setup_wecom` → `gateway.platforms.wecom.qr_scan_for_bot_info`），无需手动配置 bot 凭证。

### 飞书 / Mattermost / Matrix

- **PLATFORM_HINTS**：Matrix、Mattermost、Feishu 的系统提示覆盖
- **Feishu @mention 保留**：入站消息保留 @mention 上下文
- **Feishu 流式换行修复**：流式编辑不再前置多余空行

### Slack / Gateway 通用

- `SLACK_REACTIONS` 环境变量开关 reaction 生命周期
- MEDIA: 标签支持 PDF、document、archive 扩展名自动提取
- `require_mention_channels` 配置
- 全局 `security.allow_private_urls` 开关——解决 OpenWrt/TUN 代理/企业 VPN/Tailscale 场景（DNS 解析到 198.18.0.0/15、100.64.0.0/10 的私有 IP）。云元数据端点（169.254.169.254 等）始终阻断

## Session 存储

### 启动时自动 prune + VACUUM

`state.db` 之前无限增长。重度用户报告 384MB/982 sessions 影响性能。v2026.4.18+ 在启动时自动执行：
- 新增 `state_meta` 表记录上次运行时间戳
- 跨进程幂等（同一 `HERMES_HOME` 下只执行一次）
- 默认每天一次（`min_interval_hours=24`），清理 90 天以上已结束 session（`retention_days=90`）
- 智能 VACUUM：只有 `pruned > 0` 才执行，空清理不浪费 I/O
- 永不抛异常
