# Hermes Agent 更新日志（2026-04-09 ~ 2026-04-11）

293 个 commit，35+ 个新功能。以下为重点内容。

---

## 架构级变更

### Context Engine 插件化

**核心变更**：上下文管理从硬编码的 `ContextCompressor` 重构为可插拔的 `ContextEngine` ABC（`agent/context_engine.py`）。

```yaml
# config.yaml — 切换上下文引擎
context:
  engine: "compressor"   # 默认；设为插件名切换（如 "lcm"）
```

ABC 要求实现 3 个核心方法：
- `name` — 引擎标识
- `should_compress(prompt_tokens)` — 是否需要压缩
- `compress(messages, current_tokens)` — 执行压缩，返回新消息列表

可选方法：`on_session_start/end`、`get_tool_schemas`（引擎可暴露工具给 agent）、`update_model`（模型切换时更新）。

内置 `ContextCompressor` 现在继承自 `ContextEngine`，是默认实现。第三方插件放在 `plugins/context_engine/<name>/` 目录下。

### 统一文件同步（file_sync.py）

新模块 `tools/environments/file_sync.py`：为 SSH/Modal/Daytona 后端提供统一的文件同步，基于 mtime+size 变更追踪，支持删除检测和事务回滚。Docker/Singularity 用 bind mount 不需要。

### 后台进程监控（watch_patterns）

`terminal` 工具新增 `watch_patterns` 参数，后台进程输出匹配指定字符串时实时通知 agent：

```python
terminal(command="pytest -v", background=True, watch_patterns=["ERROR", "FAIL", "listening on port"])
```

内置速率限制（10 秒窗口最多 8 次通知），持续超载 45 秒自动禁用，防止通知风暴。

## 平台增强

### Discord

| 功能 | 说明 |
|------|------|
| **多 bot 过滤** | 消息 @了其他 bot 但没 @自己时自动跳过，解决多 bot 频道干扰 |
| **channel_skill_bindings** | 按频道自动加载不同 skill，论坛帖子继承父频道 |
| **allowed_channels 白名单** | 限制 bot 只在指定频道响应 |

### Slack

| 功能 | 说明 |
|------|------|
| **allow_bots** | 三模式（none/mentions/all），支持 bot 间通信 |
| **require_mention** | 频道内需 @提及才响应（默认 true） |
| **free_response_channels** | 指定频道免 @提及，覆盖 require_mention |

### Gateway 通用

| 功能 | 说明 |
|------|------|
| **Fast Mode** | `/fast` 命令支持 gateway，OpenAI Priority Processing |
| **统一 proxy** | Discord/Telegram 统一代理支持，macOS 自动检测 |
| **自重启** | `/restart` 命令 + SIGUSR1 信号，先 drain 在途请求再重启 |
| **WSL 检测** | WSL 环境智能 systemd 检测 |
