# Hermes Agent 更新日志（2026-04-08 ~ 2026-04-09）

## 新功能

### 结构化 API 错误分类 + 智能 Failover

**commit**: `1a3ae6ac` (#6514) | **新增文件**: `agent/error_classifier.py`（789 行）

替换了 `run_agent.py` 中分散的字符串匹配逻辑，引入统一的错误分类器。所有 API 错误现在被分类为 13 种 `FailoverReason`，每种对应不同的恢复策略：

| 错误类型 | 恢复策略 |
|---------|---------|
| `auth` | 刷新/轮换凭证 |
| `billing` | 立即切换 Provider |
| `rate_limit` | 退避等待后轮换 |
| `context_overflow` | 压缩上下文 |
| `timeout` | 重建客户端 + 重试 |
| `model_not_found` | fallback 到其他模型 |
| `server_error` | 重试 |

分类结果包含结构化的恢复提示（`retryable`、`should_compress`、`should_rotate_credential`、`should_fallback`），重试循环直接读取这些字段，不再重复解析错误。

### 统一执行层（spawn-per-call）

**commit**: `d684d7ee`

Terminal 环境架构重构。所有 6 种后端（local、docker、ssh、singularity、modal、daytona）统一为"每次命令独立 spawn `bash -c` 进程"模型：

- 初始化时捕获一次 session 快照（env vars、functions、aliases）
- 每次命令前重新 source 快照
- CWD 通过 stdout 标记（远程）或临时文件（本地）跨调用持久化

替代了之前各后端各自实现的持久 shell 方案（`persistent_shell.py` 已删除）。

---

### 工具结果持久化（3 层防护）

**新增文件**: `tools/tool_result_storage.py`、`tools/budget_config.py`

防止大型工具输出撑爆 context window 的三层机制：

| 层级 | 位置 | 机制 |
|------|------|------|
| 1. 工具内截断 | 各工具内部 | 工具自己预截断输出 |
| 2. 单结果持久化 | `tool_result_storage.py` | 超阈值（默认 100K 字符）→ 写入 `/tmp/hermes-results/`，context 里只保留 1500 字符预览 + 文件路径 |
| 3. 轮次聚合预算 | `budget_config.py` | 单轮所有工具结果合计超 200K 字符 → 最大的非持久化结果溢出到磁盘 |

`read_file` 被 pin 为阈值 `∞`，防止"持久化 → 读取 → 再持久化"的死循环。

| 修复 | commit | 影响 |
|------|--------|------|
| Session 删除改为 orphan | `a9409990` (#6513) | 删除/清理 session 时，子 session（压缩分裂产生的）不再级联删除，改为 orphan（`parent_session_id` 置 NULL），保留历史可搜 |

## 架构变化总结

1. **错误处理**：从散落在 `run_agent.py` 的 190 行字符串匹配 → 集中到 `error_classifier.py` 的结构化分类器
2. **终端执行**：从各后端各自维护持久 shell → 统一 spawn-per-call + session snapshot
3. **工具结果**：新增 3 层溢出防护，大输出不再撑爆 context window
4. **Session 管理**：删除操作从级联删除改为 orphan，保护历史数据完整性
