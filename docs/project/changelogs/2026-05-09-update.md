## 架构级变更

### 1. Provider Profile 插件化（重大架构）

`providers/base.py`（165 行） + `providers/__init__.py`（191 行）建立 **`ProviderProfile` ABC** —— 用单一声明式 dataclass 描述每个 inference provider 的 auth / endpoints / 模型目录 / client quirks / request quirks，取代散落的 20+ boolean flags。

**插件目录布局**（与 `plugins/platforms/` 镜像）：

```
plugins/model-providers/
├── openrouter/
│   ├── __init__.py    # 调用 register_provider(profile)
│   └── plugin.yaml    # name, kind: model-provider, version
├── anthropic/
├── bedrock/
├── gemini/
├── ai-gateway/
├── nous/
├── xai/
├── ...
```

**当前 28 个 bundled provider 插件**（`plugins/model-providers/` 实测）：
`ai-gateway, alibaba, alibaba-coding-plan, anthropic, arcee, azure-foundry, bedrock, copilot, copilot-acp, custom, deepseek, gemini, gmi, huggingface, kilocode, kimi-coding, minimax, nous, nvidia, ollama-cloud, openai-codex, opencode-zen, openrouter, qwen-oauth, stepfun, xai, xiaomi, zai`

**发现机制**（`providers/__init__.py:_discover_providers`）：

1. Bundled — `<repo>/plugins/model-providers/<name>/`
2. User — `$HERMES_HOME/plugins/model-providers/<name>/`（last-writer-wins，可 override 内置）
3. Legacy — `providers/<name>.py` 单文件（仅向后兼容）

每个插件 `__init__.py` 在 import 时调用 `register_provider(profile)` 完成自注册。`get_provider_profile(name)` / `list_providers()` 首次调用时触发懒加载。

**ABC 钩子**（覆盖即可加 quirk）：
- `prepare_messages(messages)` — codex 字段清洗后 / developer-role swap 前
- `build_extra_body(session_id, **ctx)` — 合并到 extra_body
- `build_api_kwargs_extras(reasoning_config, **ctx) → (extra_body_dict, top_level_kwargs)` —— OpenRouter 把 reasoning 放 extra_body，Kimi 放 top-level；split 让 transport 一次拿到正确分流
- `fetch_models(api_key, timeout)` — 默认实现走 `models_url || base_url + "/models"`，OpenRouter / Gemini 等覆盖

详见 [Smart Model Routing](concepts/smart-model-routing.md)、[Provider Transport Architecture](concepts/provider-transport-architecture.md)。

### 3. `/goal` —— 跨轮持续目标（Ralph 循环）

`hermes_cli/goals.py` 引入 `GoalManager`，把 Ralph 循环作为一等 primitive：

- 每轮结束后辅助模型 judge —— "目标是否被最后一条 assistant 响应满足？"
- 不满足 → 注入 continuation prompt 到同一 session，继续干
- 终止条件：goal 完成 / turn budget 耗尽 / 用户 `pause` 或 `clear` / 用户发新消息（preempt）
- **零 system prompt 突变、零 toolset swap** —— prompt cache 完整保留
- Judge 失败 fail-OPEN（continue），turn budget 是后盾
- 状态写 SessionDB 的 `state_meta` 表，键 `goal:<session_id>`，`/resume` 自动捡起来

CLI / gateway 共享同一个 `GoalManager`（PR #18262 / #18275 / #21287）。

斜杠命令：`/goal [text | pause | resume | clear | status]`（`hermes_cli/commands.py:103`）。

### 4. Checkpoints v2 —— 单仓重写

`tools/checkpoint_manager.py`（1638 行）+ `hermes_cli/checkpoints.py`（244 行）—— 重写状态持久化，**实质性 prune + 磁盘配额**。

**v1 痛点**：每个 workdir 一个完整 shadow git repo，不共享 objects，磁盘暴增、孤儿 repo 不被清理。

**v2 单仓布局**（`~/.hermes/checkpoints/`）：

```
store/                           # 单一 bare-ish git repo（共享 objects）
    HEAD, config, objects/       # 标准 git internals
    refs/hermes/<hash16>         # per-project 分支 tip
    indexes/<hash16>             # per-project git index
    projects/<hash16>.json       # {workdir, created_at, last_touch}
    info/exclude
.last_prune                      # 自动 prune 幂等 marker
legacy-<timestamp>/              # v1 → v2 自动归档
```

触发：每轮 `write_file` / `patch` / `terminal`（破坏性 flag）前，每会话每轮一次快照。LLM 看不到 checkpoint manager —— 它是透明基础设施，由 `checkpoints` 配置项 / `--checkpoints` flag 控制。

CLI：`hermes checkpoints` 子命令；slash：`/rollback [number]`（`commands.py:86`）。

## Cron —— `no_agent` 看门狗模式

`cron/jobs.py:498` 新增 `no_agent: bool = False`：

- `no_agent=True` 时 cron job **完全跳过 agent**，纯执行 `script` —— stdout 空 = 静默，非空 = 原样投递
- `no_agent=True` 时 `prompt` 仅作为 name hint，`workdir` 仍然作为脚本 cwd
- `cron/jobs.py:581` 强制要求 `no_agent + script` 共存（脚本 IS the job）
- `deliver=all` 路由意图（PR #21495）—— 一个 job 扇出到所有连接 channel
- 集成 watchers skill（`optional-skills/devops/watchers/` —— RSS / HTTP JSON / GitHub polling 带 watermark dedup）

## Windows 原生支持（Beta）

PR `e93bfc6c9`、`b7fe7ed7b`、`9de893e3b`、`cc38282b0`、`9c263fbf8` 等批量关闭 POSIX 假设：

- `psutil` 接管 PID / process management，添加 `_pid_exists()` 兜底（`os.kill(pid, 0)` 在 Windows 不是 no-op）
- bundled portable MinGit（不依赖 winget）
- TUI crash、kanban waitpid、AF_UNIX sandbox、`/bin/bash`、npm `.cmd` shims、cwd tracking、detach flags 全修
- Ctrl+Enter / Shift+Enter 在 Windows Terminal / WSL / SSH 插入换行
- gateway 装为 Scheduled Task（Startup folder 兜底）
- portable tooling、User env、PATH、Scheduled Task 卸载清理
- UTF-8 stdio shim（`hermes_bootstrap.py` 入口点）+ tzdata 依赖
- doctor 自动安装 Playwright Chromium
- Windows footgun checker + ruff-check + windows-footguns CI 阻断

文档：`docs(windows)` 标记 Native Windows 为 **early beta**（PR #22115），首页加 install 单行（PR #22089）。
