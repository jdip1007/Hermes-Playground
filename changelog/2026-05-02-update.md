# Hermes Agent 更新日志（2026-04-23 ~ 2026-05-02）

## 2. 自我改进闭环（Background Review Fork）重做

后台审查 fork（每轮决定保存/更新哪些 memory/skill）做了**全栈升级**：

- **rubric-based 评分**（class-first）替换以往的自由问答 "should this update"
- **active-update bias** —— 优先更新刚被 agent 加载的 skill；理解 `references/` 和 `templates/` 子文件
- **fork 继承父 runtime**（provider/model/credentials 真正传递）—— 之前 fork 会回落到默认配置
- **toolsets 限定到 memory + skills** —— fork 不能调 web/shell（防止越权）
- 后台 memory provider clean shutdown
- review summary **排除上一轮 tool messages** —— fork 看到干净的对话上下文

## 6. Session / Memory / FTS5

### Trigram FTS5 中文搜索（重大）

`hermes_state.py:128-156` 新增 `messages_fts_trigram` 虚拟表，使用 `tokenize='trigram'`：默认的 unicode61 把每个 CJK 字符切成独立 token，破坏短语匹配；trigram 创建 3-byte 滑动窗口，CJK / 泰语 / 任何脚本的子串查询都原生工作。

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS messages_fts_trigram USING fts5(
    content,
    tokenize='trigram'
);
```

### tool_name + tool_calls 入索引

`hermes_state.py:111,123,141,153` —— FTS5 索引现在覆盖 `content || tool_name || tool_calls`，可以搜 "调用过 web_search 但没找到结果" 这类语义。Schema v11 自动迁移（`hermes_state.py:440-481`）。

- Hindsight clean shutdown，配置 `HINDSIGHT_TIMEOUT` env
- Memory provider 在 mid-process `session_id` rotation 时收通知（fixes #6672）

## 11. 安全与可靠性

### Secret redaction 默认关闭（行为变更）

`agent/redact.py:60-64`：

```python
# OFF by default — user must opt in via
# `security.redact_secrets: true` in config.yaml (bridged to this env var
# in hermes_cli/main.py and gateway/run.py) or `HERMES_REDACT_SECRETS=true`
_REDACT_ENABLED = os.getenv("HERMES_REDACT_SECRETS", "").lower() in ("1", "true", "yes", "on")
```

bridge 在 `hermes_cli/main.py:176-191`。这条改动直接终结了**长期 patch 损坏 incident**：假 secret-shaped 字符串（看起来像 key）会被替换，结果是工具输出畸形、补丁应用失败。需要时显式 opt-in。

### `[SYSTEM:` → `[IMPORTANT:` 标记重命名

所有用户注入的标记从 `[SYSTEM: ...]` 改名 `[IMPORTANT: ...]`，绕开 Azure content filter（之前 Azure 把"SYSTEM"误判为提示注入）。涉及 `gateway/run.py:909-922`、`agent/skill_commands.py:440,487`、process_registry 等多处。

### Hardline 命令黑名单

`tools/approval.py:146-196` 新增 `HARDLINE_PATTERNS`（12 条 unconditional block 模式 + 47 条 DANGEROUS）。Hardline 命令**无法批准**，直接 fail-closed。`HARDLINE_PATTERNS_COMPILED` 预编译以减少冷启动开销。

### 其他

- **canonical `mask_secret()` helper** + status.py DIM 调色统一
- **paste.rs 上传**到期了走真实定时器扫
- **atomic file writes 保留 symlink**
- **Probe `/dev/tty`** 改成尝试打开（不是 `os.path.exists`）
- **WebSocket 允许非 loopback IP**（仅 `--insecure` 模式，issue #18633）
- 全局开关允许 private/internal URL 解析（受控环境用）

详见 [[security-defense-system]]。
