## 架构级新模块

### 多 Agent Kanban —— 持久化协作板（重大新架构）

`hermes_cli/kanban.py`（2677 行）+ `hermes_cli/kanban_db.py`（6286 行）+ `tools/kanban_tools.py` + `plugins/kanban/`（含 dashboard、systemd）：

把 v0.12.0 之前的"实验"重做成**持久化、多 Profile、可靠的多 Agent 工作板**。一台机器、多个 Hermes worker 自己拣任务、协作、交接、闭环。

**核心机制**：

- **Worker 生命周期**：heartbeat（`tools/kanban_tools.py:547-580` `_handle_heartbeat()`）+ TTL 过期 → reclaim → zombie detection；exit-without-complete 自动 block；retry budget 用尽 → fail-column；shutdown race fix。
- **完整性保障**：worker 必须**拥有任务**才能调用破坏性工具（task-ownership 校验）；KANBAN_GUIDANCE（`agent/system_prompt.py:34,120`）作为 lifecycle contract 注入系统提示。
- **Hallucination 恢复**：worker 声称创建/完成卡片但**实际没有**会被 dashboard 标记 `completion_blocked_hallucination`（`plugins/kanban/dashboard/plugin_api.py:217`），进入恢复流程。
- **Per-task `max_retries` override**：通过 `--max-retries` CLI flag（`hermes_cli/kanban.py:1281-1306`），任务级覆盖 dispatcher 默认值。`max_retries=1` 即首败即 block；`max_retries=3` 允许两次重试（`hermes_cli/kanban_db.py:647-652`）。
- **多 project boards**：一次安装、多块板，跨 profile 共享 board、workspace 和 worker log。
- **Dashboard 集成**：inline create 表单含 workspace kind + path、home-channel 通知开关、tenant 过滤、reject 直接 → running 状态变更（必须 ready → running）。
- **诊断引擎**：generic diagnostics engine 统一识别 task 困扰信号。

> 详见新页面 [[multi-agent-architecture]] 的"Kanban 持久协作板"小节。

### `/goal` —— Ralph Loop 一等公民

`hermes_cli/goals.py`（762 行）让 agent 锁定一个**跨轮持久目标**，每轮结束跑 judge 评分（`agent/system_prompt.py` 注入 goal block，`hermes_cli/goals.py:298` `_goal_judge_max_tokens()`），未达成则继续推进直到 DONE。注册：`hermes_cli/commands.py:105`。

**`/subgoal <text>`**（`hermes_cli/commands.py:107`，goals.py:156、79、86）：mid-loop 把额外成功条件**追加**到正在跑的 `/goal` 上。judge 会把新 criteria 一起算进 DONE 判定，不必重启循环。

### Checkpoints v2 —— state 持久化重写

`tools/checkpoint_manager.py`（1638 行）：v2 layout（line 30、213、340-362）配真正的 pruning（`max_snapshots`，line 966）+ disk 上限（"Checkpoint store exceeded...MB"，line 1097），杜绝 orphan shadow repo。

### Gateway Auto-Resume —— 会话穿越重启

Gateway 在 agent 跑到一半被 bounce、`/update` 重启、源文件 reload 后，会话会自动恢复（`gateway/run.py:3543,3565` "Scheduled auto-resume"、`:5620` `mark_resume_pending`）。

### Provider 插件化 —— ProviderProfile ABC

`providers/base.py:39` 定义 `ProviderProfile` ABC，`plugins/model-providers/` 收纳 20+ provider 作为可拔插件。第三方 provider 无需改 core 即可接入。

### LSP 语义诊断的前哨：post-write delta lint

`tools/file_tools.py:1045,1061,1171`：`write_file` + `patch` 写完后自动跑 Python / JSON / YAML / TOML 语法检查，**只回报新增的错误**给 agent。（v0.14 升级成全量 LSP semantic diagnostics —— 详见下一版 changelog。）

### `no_agent` cron 模式 —— script-only watchdog

`hermes_cli/cron.py:96,181` + `tools/cronjob_tools.py:322-372`：cron 任务可以**完全跳过 agent**，只跑脚本。空 stdout 静默；非空 stdout 原样投递。低成本轮询/监控用例。

## 多 Agent 控制面 —— `/handoff`、`/steer`、`/queue`

`hermes_cli/commands.py:82,101,103`：

- **`/handoff <profile>`** — 主动把当前会话交到另一 profile
