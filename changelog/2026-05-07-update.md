# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-07）

跨两个版本：**v0.12.0 (v2026.4.30) The Curator Release**（1096 commits / 550 PRs）+ **v0.13.0 (v2026.5.7) The Tenacity Release**（864 commits / 588 PRs）。本次同步覆盖 v2026.4.23（上次 wiki 跟踪点）之后的全部公开变更。

---

## 一、v0.12.0（2026-04-30）— Curator Release

> 主题：**Hermes 自我维护**。后台 Curator 自动评分 / 整合 / 修剪技能库；Self-improvement loop 升级为 class-first；新增 4 个推理 Provider；第 18 / 19 个消息平台；ComfyUI、TouchDesigner-MCP 默认内置。

### 1.1 Autonomous Curator（已部分覆盖于 04-29 update，此处补全）

- `agent/curator.py`（869 行）+ `hermes_cli/curator.py` —— 后台 agent，挂在 gateway 的 cron-ticker 上跑（默认 7 天周期）。
- 通过 `auxiliary.curator` 配置驱动；可在 `hermes model` 选 curator 模型，dashboard 可管理。
- `hermes curator status` 按使用次数排序（most-used / least-used）。
- 双重保护：bundled / hub-installed skills 永不修改（按 frontmatter slug 匹配，详见 #20194）。
- 每次运行写 `logs/curator/run.json` + `REPORT.md`，归档可分类为「整合的」vs「修剪的」。

### 1.2 Self-improvement loop 升级

- 改造为 **class-first**（rubric-based 而非自由形式）。
- **active-update biased**：偏好 agent 刚加载过的 skill。
- 处理 `references/` / `templates/` 子文件。
- 正确继承父 agent 的 live runtime（provider / model / 凭证真正传过去）。
- 限制在 memory + skills toolset，避免膨胀。
- Memory provider 干净关闭；prior-turn tool message 排除在 summary 外。

### 1.3 新增 Provider（4 个）

| Provider | 类型 | 来源 |
|----------|------|------|
| **GMI Cloud** | first-class（salvage #11955） | `plugins/model-providers/gmi/__init__.py` |
| **Azure AI Foundry** | 自动检测 | `plugins/model-providers/azure-foundry/` |
| **MiniMax OAuth** | PKCE browser flow（salvage #15203） | 已在 04-29 covered |
| **Tencent Tokenhub** | salvage #16860 | provider 配置 |
| **LM Studio** | 升级为 first-class（不再仅 alias） | `hermes doctor` 检查 + reasoning transport + live `/models` |

源码验证：`plugins/model-providers/` 目录下 14 个 provider 插件（gmi / kilocode / stepfun / openai-codex / alibaba / xiaomi / xai / azure-foundry / copilot-acp / kimi-coding / zai / gemini / custom / ollama-cloud），全部基于 `providers/base.py:25` 的 `ProviderProfile` dataclass。

### 1.4 第 18 / 19 个消息平台

- **Tencent 元宝 (Yuanbao)** —— 第 18 个原生平台（已在 04-29 covered）。
- **Microsoft Teams** —— 第 19 个，**首个插件实现**的平台。Gateway 现在是插件 host。
  - 源码：`plugins/platforms/teams/`

### 1.5 Spotify + Google Meet

- **Spotify**：7 个原生工具（play / search / queue / playlists / devices）+ PKCE OAuth + 交互式 setup wizard + 内置 skill。
  - 源码：`plugins/spotify/__init__.py` (66 行) + `client.py` (435 行) + `tools.py` (454 行)
- **Google Meet plugin**：参加会议、转写、说话、跟进。Realtime OpenAI transport + Node bot server。
  - 源码：`plugins/google_meet/`（含 `node/` 子进程 + `realtime/` 实时管线）

### 1.6 Skill 集成扩展

- **ComfyUI v5**：官方 CLI + REST API + 硬件 gated 本地安装；从 optional 移到 **bundled by default**。
- **TouchDesigner-MCP**：默认 bundled，扩展 GLSL / post-FX / audio / geometry + 9 篇参考文档。
- **Humanizer** skill：移植文本清洗，剥离 AI-isms。
- **claude-design** HTML artifact skill + design-md（Google DESIGN.md spec）+ airtable salvage + `skill_manage` 在 `external_dirs` 编辑 + 直接 URL 安装 skill + `/reload-skills` 斜杠命令（已在 04-29 covered）。

### 1.7 CLI / 一次性模式

- **`hermes -z <prompt>`**：非交互式 one-shot，支持 `--model` / `--provider` / `HERMES_INFERENCE_MODEL`。
- **`hermes update --check`**：preflight。
- **opt-in pre-update HERMES_HOME backup**。

## 二、v0.13.0（2026-05-07）— Tenacity Release

> 主题：**Hermes finishes what it starts**。Multi-agent Kanban 持久看板（heartbeat / reclaim / zombie 检测 / per-task 重试 / 幻觉恢复）；`/goal` 跨轮目标锁定（Ralph loop）；Checkpoints v2 重写状态持久化；Gateway 自动恢复中断 session；Cron `no_agent` 看门狗模式；8 个 P0 安全闭环；Google Chat（第 20 个平台）；Provider 变成可插拔 surface；7 个 i18n 语言。

### 2.1 Multi-agent Kanban —— 持久协作板

**核心源码**：
- `tools/kanban_tools.py`（871 行）
- `hermes_cli/kanban.py`（2184 行）
- `hermes_cli/kanban_db.py`、`hermes_cli/kanban_diagnostics.py`、`hermes_cli/kanban_specify.py`

**特性**：

| 机制 | 说明 |
|------|------|
| **Multi-profile by design** | 一次安装，多块看板（multi-project boards） |
| **Heartbeat + reclaim + zombie 检测** | `kanban_heartbeat` 工具同时延 claim TTL 和记录 worker 心跳；阻塞超过 `DEFAULT_CLAIM_TTL_SECONDS` 自动 reclaim。源码 `tools/kanban_tools.py:317`（`_handle_heartbeat`） |
| **Hallucination gate** | worker 创建的卡片走幻觉门控 —— `kanban_complete` 阻塞包含 `created_cards` 但未通过 `kanban_create` 调用的关闭（`tools/kanban_tools.py:263`） |
| **Per-task max_retries override** | 单卡级别覆盖默认重试预算（#21330） |
| **Auto-block on incomplete exit** | worker 不带 complete 退出 → 自动 block + shutdown race fix（#21214） |
| **Generic diagnostics engine** | task distress 信号通用诊断引擎（#20332） |
| **Tool ownership 强制** | worker 只能在已认领的 task 上做破坏性操作（#19713） |
| **Cross-profile sharing** | board / workspace / worker logs 跨 profile 共享 |
| **`/kanban` 斜杠命令** | 已文档化（#19584） |
| **`/specify` 命令** | 辅助 LLM 帮 triage task 写完整描述（`hermes_cli/kanban_specify.py`，#21435） |

**新优化 skill**：`kanban-video-orchestrator` (@SHL0MS, #19281)。

详见 [[multi-agent-architecture]]。
