# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-13）

跨越两个 release：**v0.12.0** (`73bf3ab`，2026-04-30) → **v0.13.0**（当前 `pyproject.toml` 声明）。595 个 commit。核心主题：

1. **Microsoft Teams / LINE / Google Chat 三个新 messaging platform**（全部以插件形式接入）
2. **Curator 大幅扩展**：archive/prune/backup/rollback 子命令、状态机分裂、bump_use 覆盖、统一到 auxiliary 入口
3. **Per-turn file-mutation verifier footer** + **LSP semantic diagnostics** + **in-process delta lint**
4. **`/handoff` 跨平台 session 转移** + **Telegram `/topic` DM 多 session 切换**
5. **Piper 本地 TTS** + **命令型 TTS provider 注册表** + **gateway 音频路由集中化（含 FLAC）**
6. **Prompt cache：1h cross-session 前缀 + 系统提示 byte-static**
7. **i18n 16 语种覆盖 gateway + dashboard**
8. **Kanban 调度系统大规模 hardening**（dispatcher、heartbeat、ACL、UI）
9. **Supply-chain advisory checker** + **lazy-install / tiered install framework**
10. **`ctx.llm` 让插件直接调用宿主模型** + **bundled hermes-achievements + plugin platforms**

## Curator 系统扩展

### 大幅扩容 & 统一到 auxiliary（0da66e8）

| 模块 | v2026.4.23 | v0.13.0 | 增量 |
|------|-----------|---------|------|

`fix(curator): unify under auxiliary.curator (hermes model, dashboard)` (`0da66e8`) —— curator 不再走专用 client，统一到 `auxiliary.curator.*` 配置入口（和 dashboard、`hermes model` 共享相同的 aux 路径）。

### 新增子命令（436672d）

```bash
hermes curator archive <skill>      # 手动归档
hermes curator prune                # 物理移除已 archived skill（带 backup）
hermes curator backup               # 显式打 backup 快照
hermes curator rollback             # 还原最近 backup
hermes curator list-archived        # 查看归档列表
```

`archive` ↔ `prune` 分离：archive 是可逆状态变更，prune 真删（先 backup 再删）。

### 状态机分裂（8b290a5）

`active → stale → archived` 中 `archived` 进一步分为 `consolidated`（被合并到别的 skill）vs `pruned`（无价值清除），用 model + heuristic 联合分类，便于统计和审计。

### `bump_use()` 接入更多调用点（4178ab3 + ae8930a）

skill invocation 和 preload paths 都走 `bump_use()`，连 `skill_view` 工具调用也算"使用"——闲置判定更准。

### Pinned skill 边界调整（b10e38e）

> Pin protects against **deletion only, not edits**.

之前 pin 锁死整个 skill 不让改，导致修 bug 都要 unpin。现在 pin 仅保护**删除路径**，编辑允许——更实用。

### 其他

- 嵌套 archive 子目录扫描（564a649）让 restore 找得到深层归档
- `hermes curator status` 显示 most-used / least-used 列表（d60a991）
- rename block 时建议 `hermes curator pin` 防自动整合（7312f7e）
- 默认 config seed-on-update + 自动建 `logs/curator/` 目录 + `fire` lazy import（e8e5985）
- skill index walk 和 `skills_tool` 全部跳过 `.archive/`（eda1d51 + a845177）
- **frontmatter name 保护 hub skills**（68e4664）：通过 name 而非 path 识别 hub skill，避免重命名后旁路

## 文件修改与 LSP

### Per-turn file-mutation verifier footer（PR #24498，c594a23）

每轮 agent 调用结束时，user 一侧追加一段 footer 列出**本轮 write_file/patch 失败但 agent 没修好**的文件，模型下一轮即可看到。

- 配置 `display.file_mutation_verifier`（bool，默认 True）
- env `HERMES_FILE_MUTATION_VERIFIER`（0/1/false/true）覆盖配置
- 判定 landed 走 `file_mutation_result_landed()`，看 tool 结果里的 landed 状态而非 `is_error` flag（`fix(da0ddbf)`）—— compaction 等场景里 patch 已落盘但拿不到 tool result，照样算成功

源码 `run_agent.py:5331-5395`。

### LSP 语义诊断（PR #24168，83b9389）

`write_file` / `patch` 落盘后跑 LSP（pyright / tsc / shellcheck）拿语义诊断，作为独立 channel 返回给 agent（不是 error），让模型能看到"语法 OK 但类型不对"这类问题。

实现：`tools/file_operations.py:929-991`，`_snapshot_lsp_baseline()` 在写之前抓 baseline，写后只回 diff 出来的新 diagnostic。

后续 follow-up:
- typescript SDK install 修复（29c9ff9）
- tsc-missing skip（29c9ff9）
- shellcheck warning 抑制（29c9ff9）
- `'lsp'` 加入 `_BUILTIN_SUBCOMMANDS` 跳过 plugin discovery（71c6dd0）

### In-process delta lint（PR #20191，5168226）

`write_file` 和 `patch` 后**同步**跑 in-process linter：JSON / YAML / TOML / Python。返回**差量** diagnostic（只展示新引入的问题，不刷屏老问题）。

详见 [[agent-loop-and-prompt-assembly]]。

## Prompt Cache

### Cross-session 1h 前缀（PR #23828，7b76366）

`prompt_caching.cache_ttl: "1h"` 让 Claude 在 Anthropic / OpenRouter / Nous Portal 三条路径上共享 1 小时 cache 前缀，跨 session 命中。写成本 2x（5m 是 1.25x），长 session 摊销下来更省。

### 系统提示 byte-static（PR #24778，b06e999）

> kill long-lived prefix layout — system prompt is now byte-static within a session

弃用"保留 prefix 跨重建稳定"的旧策略，改为**一个 session 内 system prompt 字节完全一致**——工具索引、技能索引、内存快照都在 session 启动时冻结。结果：
- 不需要复杂的 prefix preservation patch
- 缓存命中等价于"字节相同"，调试直接
- 跨 session（1h 模式）也能复用

### Nous Portal Qwen 缓存路径修复（7993e03 + 2a18b62）

- 之前走错 cache pathway，改成走 Portal-Claude cache pathway（#24151）
- TTL=1h 在 Alibaba 上游不支持（只 5m），降级（#24702）

详见 [[prompt-caching-optimization]]。

`tools/tts_tool.py:309-470` —— 用户可在 `~/.hermes/config.yaml` 声明任意 `type: command` 命名 provider：

```yaml

      type: command
      command: "piper -m ~/model.onnx -f {output_path} < {input_path}"
```

## 国际化

### i18n 16 语种（PR #22914，c391684）

`agent/i18n.py` 引入轻量框架，`locales/*.yaml` 16 个语种：

```
af, de, en, es, fr, ga, hu, it, ja, ko, pt, ru, tr, uk, zh, zh-hant
```

仅覆盖**最高影响**的静态字符串：approval 提示、若干 gateway 斜杠命令回复、restart-drain 通知。Agent 输出、日志、工具结果、slash command 描述**保持英文**（避免增量翻译噪声）。

```python
from agent.i18n import t
t("approval.choose_long")              # 用当前 language
t("gateway.draining", count=3)         # 含 {count} 插值
t("approval.choose_long", lang="zh")   # 显式 override
```

解析顺序：`lang=` 参数 → `HERMES_LANGUAGE` env → `display.language` config → `"en"`。

i18n 同步覆盖 **web dashboard**（PR #22914 同一批）。

## Kanban 调度系统

广义"任务管理 + dispatcher"——本周期内大规模 hardening：

| 模块 | 行数 | 关键文件 |
|------|------|---------|
| `hermes_cli/kanban.py` | 2252 | CLI 入口 |

| `tools/kanban_tools.py` | 1139 | Worker agent 工具（仅在 `HERMES_KANBAN_TASK` 设置时进入 schema） |
| `hermes_cli/kanban_diagnostics.py` | 776 | 健康检查 |
| `hermes_cli/kanban_specify.py` | 266 | 任务规格化 |

### 设计要点（`tools/kanban_tools.py` docstring）

> Tools 只在 dispatcher 运行时（`HERMES_KANBAN_TASK` 设置）注册，普通 `hermes chat` 看不到。
> 1. **Backend portability**：worker 在 Docker/Modal/SSH 里运行时不一定能跑 `hermes kanban`，但工具直接读 `~/.hermes/kanban.db`
> 2. **No shell-quoting footguns**：metadata 复杂结构走 tool args
> 3. **Better errors**：tool 返回结构化 JSON

### 主要变更

- **Orchestrator board tools**（236bfd5 + 2704e7b + 6e5c49b）：board routing tools 限制给 orchestrators，Step-0 profile discovery（弃用 hardcoded specialist 名单）
- **Dispatcher 修复**：spawn 用 `sys.executable -m hermes`（d3db672 + 62b1c74）、`HERMES_HOME` 注入 worker env（5af315c）、`HERMES_KANBAN_BOARD` 在 chat boot 时 pin 防 subprocess drift（b22b3f5）
- **Heartbeat / stale lock**：stale 阈值上调（0cc6304）、stale claim **续期** 而非 kill live worker（88588b6）
- **ACL / ownership**：dashboard PATCH 拒绝直接转 running（6b3efce）、worker 任务 ownership 强制（d3db672 → d3b22b7）
- **Dashboard QOL**：multi-card drag、shift-click range、column select-all、bulk action 乐观 UI（a88f201 + 3df7e30 + 0ea234e）
- **通知去重**：blocked/gave_up 去重（a96dd54 + 861ce7c）、gateway create auto-subscribe 路由到 explicit board（f6d4f3c）
- **`stranded_in_ready` 诊断**（3b122cc）：无人 claim 的任务诊断输出
- **`kanban_block` on iteration-budget exhaustion**（2b3bf17）：worker 耗尽 budget 时主动 block 避免协议违规
- **Per-platform home-channel toggle**（#19864、#19916）
- **Send_message gating**：kanban worker 可以 call send_message（8ac998c）

## 安全

### Supply-chain advisory checker（PR #24220，c1eb2dc）

`hermes_cli/security_advisories.py` —— 检测**已知被入侵的** Python 包版本（如 2026 年 5 月的 Mini Shai-Hulud 蠕虫毒化的 `mistralai 2.4.6`），通过 `importlib.metadata.version()` 一次调用拿到，启动时跑：

- 没装中招版本 → 无声
- 装了 → CLI banner 一行 + `hermes doctor` 详细指南
- 用户 ack 后存 `config.security.acked_advisories`，幸存于重启
- 只用 stdlib，即使主体 Hermes import 失败也能跑

调用点：`hermes doctor` / CLI 启动 banner / gateway 启动。

### Lazy-install / tiered install fallback（PR #24220，c1eb2dc）

`tools/lazy_deps.py` —— 按需安装平台扩展（slack/matrix/dingtalk/feishu 等）。失败时分级 fallback。

### Approval DELETE pattern DOTALL（80374d4）

修复审批 regex 用 DOTALL flag 允许换行 bypass 的漏洞。

### Subprocess shell=True 缩减（d6c9711）

降低 shell injection 攻击面。

### Sudo 限制（9520a1c + 976d8e2）

terminal backend 屏蔽 `sudo -S` 密码猜测；catch `sudo --stdin / --askpass / shell privilege flags`。

### QQBot guild ACL（d69a0b2）

guild messages 和 guild DM 也走 ACL 检查，堵 allowlist bypass。

### Dashboard plugin API auth（ec9329e + ae4b09c）

所有 dashboard plugin API 路由强制 auth。

### Quick command env sanitize + output redact（f6736ce）

quick commands 路径上 env sanitize + 输出 redact，防止误泄环境变量。
