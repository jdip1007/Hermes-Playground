# Hermes Agent 更新日志（2026-04-30 / v0.12.0）

发布 **v0.12.0 / v2026.4.30**，1096 个 commit、550 PR、217k 行新增、213 名贡献者。代号 **The Curator Release** —— Hermes 现在能**自我维护技能库**。

## 架构级变更

### 自治 Curator：技能库的"园丁"（重大新模块）

`agent/curator.py`（1781 行）+ `hermes_cli/curator.py`（598 行）把上一版引入的 Curator 升级成**真正的后台 agent**：

- **运行机制**：挂载到 gateway 的 cron-ticker，**7 天默认周期**（`DEFAULT_INTERVAL_HOURS = 24 * 7`，`agent/curator.py:56`），每周自动跑一次"评级 → 整合 → 归档"。
- **状态机**：`active → stale → archived`，重新被调用回到 `active`。Curator 还会用模型 + 启发式把已归档技能区分为 **consolidated（被合并）** 还是 **pruned（无用淘汰）**，落入 `split_archive_decisions()`（`agent/curator.py:498-611`）。
- **统一配置面**：现在归在 `auxiliary.curator` 之下（`agent/curator.py:1606-1614`、`hermes_cli/config.py:3895`），从 `hermes model` 里挑模型、从 dashboard 里管。
- **CLI 子命令大扩**（`hermes_cli/curator.py:487-565`）：`status / run / pause / resume / pin / unpin / restore / list-archived / archive / prune / backup / rollback`。
- **不变量（仍生效）**：永不触碰 bundled/hub 技能；只归档不删除；pinned 跳过所有自动转换；用 aux client，绝不污染主 session 的 prompt cache。
- **技能使用计数**：`bump_use()`（`tools/skill_usage.py:416`）被接进了 `skill_view` / preload / cron / skill_bundles / skill_commands，curator 能拿到真实活动度数据再排序。
- **`_pinned_guard()`**（`tools/skill_manager_tool.py:137`）：拦截 `skill_manage` 的 create/update/archive/delete 路径，扩展 Curator 的 "pinned 不可写" 不变量；连主 agent 也无法绕过。
- **每轮报告**：`logs/curator/run.json` + `REPORT.md`。

> 详见 [[skills-system-architecture]]。

### Self-improvement loop 大幅升级

后台审查 fork（Hermes 自我改进的核心：每轮结束后判断"要不要存技能/记忆"）升级为**类目驱动**而非自由形式：

- **Class-first rubric** — 把"应该 update 吗"换成"按类别给分"
- **Active-update bias** — 偏向更新 agent 刚加载的那个技能；处理 `references/` + `templates/` 子文件
- **Fork 继承父进程 live runtime** — provider、模型、credentials **现在真的会传播**（之前是默认值）
- **Scoped toolsets** — review fork 仅限 memory + skills（无 shell、无 web，防止 sprawl）
- **Clean shutdown** — memory provider 正常退出
- **Clean context** — 之前轮次的 tool message 不再灌进 summary

### 平台适配器插件化进一步成熟（紧承 v0.11.0）

`PlatformRegistry` + `PluginContext.register_platform()` 在上版搭好，本版做了大量"插件平台和内置平台完全等价"的扫尾，并交付了**第一个由 plugin 模式接入的官方平台**：**Microsoft Teams**（`plugins/platforms/teams/`，作为 plugins/teams_pipeline 的入口）。

`gateway/platforms/` 同时新增了原生 **腾讯元宝 Yuanbao**（`yuanbao.py` + `_media.py` + `_proto.py` + `_sticker.py`），作为第 18 个平台。

> 详见 [[messaging-gateway-architecture]]。

## 性能与运维

### 冷启动性能 — 可见 TUI 冷启动砍 ~57%

- **lazy agent init**
- **lazy import**：OpenAI / Anthropic / Firecrawl / account_usage（`cli.py:96-97` 显式注释了 OpenAI SDK ~230ms 冷启延迟）
- **mtime-cached `load_config()`**
- **memoized `get_tool_definitions()`** 配 TTL-cached `check_fn`
- **precompiled 危险命令正则**

### Prompt Cache TTL 可配置

`prompt_caching.cache_ttl`（5m 默认，1h 可选 —— bursty session 保持 cache 暖时省钱）。代码：`agent/prompt_caching.py:51` + `agent/agent_init.py:475-485`，配置 stub `hermes_cli/config.py:820-822`。

### 默认 Secret Redaction 翻转为 OFF（本版）

防止"假密钥子串误伤工具输出"的长期 bug。**注：v0.13.0 又翻回 ON by default**（详见 v0.13 changelog 中的安全 wave）。

### `hermes -z` one-shot + `hermes update --check`

- `hermes -z <prompt>` 非交互式，配 `--model` / `--provider` / `HERMES_INFERENCE_MODEL`
- `hermes update --check` preflight
- opt-in 预升级 `HERMES_HOME` backup

### Gateway 媒体打通

- 跨 Telegram/Discord/Slack/Mattermost/Email/Signal 原生多图发送
- 集中化音频路由 + FLAC 支持 + Telegram document fallback
