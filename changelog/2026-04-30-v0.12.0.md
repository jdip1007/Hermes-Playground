# v0.12.0 (v2026.4.30) — The Curator Release

**1,096 commits · 550 merged PRs · 1,270 files changed · 217,776 insertions**

> 自治维护版本 — Hermes 开始**自己维护自己**。后台 Curator 在独立 cron ticker 上自动评分 / 合并 / 淘汰技能库；self-improvement loop 大幅升级；4 个新 inference provider；18-19 个平台（Yuanbao + Teams）；Spotify 原生工具、Google Meet 插件；ComfyUI 和 TouchDesigner-MCP 从 optional → bundled；TUI 冷启动减少约 57%。

## ✨ 头牌新增

### 1. Autonomous Curator — 自治技能维护代理

**`hermes curator` 升级为后台代理**，在 gateway 的 cron ticker 上每 7 天自动跑。
- **per-run 报告**：`logs/curator/run.json` + `REPORT.md`
- **`hermes curator status`**：最近 run 摘要 + 最常用 / 最少用 skill 排行
- **统一在 `auxiliary.curator` 槽位**：`hermes model` 里直接选 curator 模型，dashboard 配置
- **归档分类**：consolidated（被另一个 skill 吞并）vs pruned（独立淘汰），由模型 + 启发式联合判定
- 严格保护 bundled / hub-installed skill，pinned skill 阻止 `skill_manage` 写入

源码：`agent/curator.py`（1781 行）、`agent/curator_backup.py`（693 行）、`hermes_cli/curator.py`（598 行）。

### 2. Self-improvement loop — 全面升级

per-turn background review fork（Hermes 自我提升的核心）：
- **rubric-based grading**（取代之前自由文本「该不该更新」）
- **active-update bias**：偏向更新刚加载过的 skill
- **`references/` / `templates/` 子文件感知**
- **runtime inheritance**：fork 实际继承父 agent 的 provider/model/credentials
- **scoped toolsets**：fork 限定 memory + skills（不能用 shell / web）
- **clean shutdown** + **clean context**

### 3. 平台 / Provider 大扩展

| 类别 | 新增 | 备注 |
|------|------|------|
| **新 provider** | GMI Cloud、Azure AI Foundry、LM Studio（first-class）、MiniMax OAuth、Tencent Tokenhub | LM Studio 从 alias 升级 |
| **Pluggable gateway platforms** | `gateway/platform_registry.py` 扩展 | 网关变成插件 host |

| **Google Meet 插件** | `plugins/google_meet/` | 加入会议、转录、说话、跟进 |

### 4. Skills 集成（major）

- **ComfyUI v5**：从 optional → **built-in by default**，official CLI + REST + hardware-gated 本地安装
- **TouchDesigner-MCP**：从 optional → **bundled by default**，扩展 GLSL / post-FX / audio / geometry / 9 个新引用文档
- 新 skill：Humanizer（剥离 AI-isms）、claude-design、design-md、airtable、pretext、spike + sketch

### 5. 冷启动 ~57% 削减

- 懒导入 OpenAI / Anthropic / Firecrawl / account_usage
- `load_config()` 和 `read_raw_config()` 用 mtime cache
- `get_tool_definitions()` memoize + `check_fn` 结果 TTL cache
- DANGEROUS_PATTERNS / HARDLINE_PATTERNS 预编译

### 6. Configurable prompt cache TTL（PR #15065）

`prompt_caching.cache_ttl` 配置项（5m 默认，1h 可选）。之前是硬编码。详见 [[prompt-caching-optimization]]。

### 7. Remote model catalog manifest（PR #16033）

OpenRouter + Nous Portal 模型 catalog 从远端 manifest 拉，新模型上架不用等 release。

### 8. Vercel Sandbox 执行后端

`execute_code` / terminal 新后端，跑 Vercel 沙箱。

### 9. Native multimodal image routing（PR #16506）

图片按模型实际声明的 vision capability 路由，不再按 provider 默认值。

### 10. 安全 — Secret redaction 默认关闭

**注意**：v0.12 默认关掉了 redaction（PR #16794，减少 patch corruption），v0.13 又默认打开（P0 closure，PR #21193）。
