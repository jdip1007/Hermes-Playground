# v0.13.0 (v2026.5.7) — The Tenacity Release

**864 commits · 588 PRs · 829 files · 128,366 insertions · 282 issues closed (13 P0, 36 P1)**

> Tenacity 版本 — Hermes 现在会**把开始的事做完**。
> Kanban 以多 Agent 持久看板的形态重新上线（心跳、reclaim、僵尸检测、incomplete-exit auto-block、per-task 重试、幻觉门）。`/goal` 把 Ralph loop 升格为一等公民。Checkpoints v2 重写状态持久化。Gateway 会在重启后自动恢复被打断的对话。Cron 长出 `no_agent` watchdog 模式。安全大扫除关闭 8 个 P0：redaction 默认 ON、Discord 角色白名单 guild-scope、WhatsApp 默认拒陌生人、`auth.json` / MCP OAuth 关 TOCTOU。Google Chat 成为第 20 个平台。Provider 变成插件 surface。静态 i18n 出 7 种 locale。

## ✨ 头牌新增

### 1. Multi-Agent Kanban — 持久任务板

post-revert 重写版（之前 v0.12 那次被回滚）。从一开始就**为多 profile 设计**。

| 模块 | 文件 | 行数 |
|------|------|------|
| CLI 入口（38 个子命令） | `hermes_cli/kanban.py` | 2252 |
| SQLite 状态层 | `hermes_cli/kanban_db.py` | 4839 |
| Worker 健康诊断 | `hermes_cli/kanban_diagnostics.py` | 776 |
| Agent 内 kanban_* 工具（9 个） | `tools/kanban_tools.py` | 1139 |
| systemd 分发器 | `plugins/kanban/systemd/hermes-kanban-dispatcher.service` | — |
| Dashboard 插件 | `plugins/kanban/dashboard/` | — |
| Worker / Orchestrator skill | `skills/devops/kanban-{worker,orchestrator}/` | — |

可靠性保证：心跳 + reclaim + zombie 检测 + auto-block on incomplete exit + 每 task `max_retries` + **幻觉门**（worker 声称创建任务时校验数据库存在性）+ 统一 failure counter（spawn/timeout/crash 同一路径）。

worker task ownership 在 destructive tool 调用时强制（`tools/kanban_tools.py:115` `_enforce_worker_task_ownership`）。

详见 [[multi-agent-architecture]] 第四节。

### 2. `/goal` — Ralph loop 持久目标

`hermes_cli/goals.py`（593 行）。给 Agent 一个跨 turn 的目标，每轮结束后由 judge 模型裁决是否完成；未完成自动注入续作 prompt。

- 状态持久化到 `SessionDB.state_meta["goal:<session_id>"]`，`/resume` 自动恢复
- **不改系统 prompt / toolset → prompt cache 完整**
- judge 失败 fail-OPEN（continue），turn budget 兜底（默认 20）
- 用户新消息抢占续作；连续 3 次 judge parse 失败自动暂停

### 3. `video_analyze` 新工具

`tools/vision_tools.py:1168` `video_analyze_tool`。native video understanding，跑在 Gemini 等多模态模型上。(@alt-glitch, PR #19301)

### 4. xAI Custom Voices — 声音克隆 TTS

TTS provider 支持 voice cloning。(@alt-glitch, PR #18776)

### 5. 7 个新 locale

`zh / ja / de / es / fr / uk / tr` 的 gateway + CLI 静态消息。Docs 站新增 zh-Hans locale。

### 6. Google Chat — 第 20 个平台

`plugins/platforms/google_chat/` — Cloud Pub/Sub pull + Chat REST + 每用户 OAuth 文件附件。同时新增 `env_enablement_fn` / `cron_deliver_env_var` 两个 platform-plugin hooks，IRC 和 Teams 一并迁移到这套机制。

### 7. Sessions auto-resume after restart

gateway 重启 / `/update` / 源文件 reload 后，被打断的会话**自动续上**。详见 [[gateway-session-management]] 的 Durability 章节。

### 9. Checkpoints v2

`tools/checkpoint_manager.py`（1638 行）— **单一共享 shadow git store** 取代每个 working dir 一份独立 shadow repo。git 对象去重，多个 worktree 总成本接近零。新增真实 pruning + 磁盘 guardrails。auto-migrate 旧的 per-project shadow 到 `legacy-<ts>/`。

```
~/.hermes/checkpoints/store/
├── HEAD, config, objects/
├── refs/hermes/<hash16>
├── indexes/<hash16>
├── projects/<hash16>.json
└── info/exclude
```

### 10. Post-write delta lint

`tools/file_operations.py:307-393` — `write_file` 和 `patch` 后**自动跑内嵌 linter**：Python / JSON / YAML / TOML 都有 `_lint_*_inproc` 实现，shell linter fallback。语法错误立即冒到 agent 视野，不会顺着 pipeline 飘下去。

### 11. Cron `no_agent` 模式

PR #19709。cron job 可以完全跳过 LLM，只跑 shell；非空 stdout 直接投递（watchdog 模式）。详见 [[cron-scheduling]]。

### 12. 平台 allowlist 全覆盖

`allowed_channels` / `allowed_chats` / `allowed_rooms` 在 Slack / Telegram / Mattermost / Matrix / DingTalk 上统一。(PR #21251, salvage #7401)

### 13. Provider 变成插件 surface

`providers/base.py` `ProviderProfile` ABC + `plugins/model-providers/`（29 个内置 provider 插件，详见 [[smart-model-routing]] 的 provider 列表）。第三方 drop in 不动核心。

### 14. API Server long-term memory per session

`X-Hermes-Session-Key` header 给 memory provider 一个稳定的 session 标识。(#20060)

### 15. MCP 升级

- SSE transport（PR #21227，salvage #19135）

- `_wait_for_lifecycle_event` 周期 keepalive（#20209）

### 16. Curator 子命令扩展

- `hermes curator archive / prune / list-archived`（手动）
- `hermes curator run` 改为同步（直接看输出）

### 17. ACP — `/steer` 和 `/queue`

VS Code / Zed / JetBrains 现在可以从编辑器内向运行中的 agent 注入 nudge 或排队 follow-up。原子 session 持久化 + reasoning metadata 跨重启保留。(@HenkDz)

新生命周期 hook，让插件在 LLM 输出抵达对话前 reshape / filter。第一个非 None 字符串胜出。典型用途：context 压缩、内容过滤、人格转换。(PR #21235)

### 23. Nous OAuth 跨 profile 共享

shared token store：在一个 profile 登录 Nous，所有 profile 都继承同一个 session。
