# Hermes Agent 更新日志（2026-05-16 / v0.14.0）

发布 **v0.14.0 / v2026.5.16**，808 commits、633 PR、165k 行新增、545 issue 关闭（**12 P0 / 50 P1**）、215 贡献者。代号 **The Foundation Release** —— "Hermes 安装在哪都能跑，只装你真用得到的东西"。

## 安装与运行时基座

### `pip install hermes-agent` —— 正式登 PyPI

`pyproject.toml:6`：`name = "hermes-agent"`、`version = "0.14.0"`，wheel 配置 `pyproject.toml:214-227`。Ink TUI bundle + shell launcher 都打进 wheel。无需 clone 仓库或跑安装脚本。

### Native Windows 支持（早期 beta）

`scripts/check-windows-footguns.py` + `pyproject.toml:60,66`（tzdata + psutil for cross-platform PID）：原生 `cmd.exe` + PowerShell 跑得通，不再依赖 WSL。完整 PowerShell installer 含 MinGit 自动安装、Microsoft Store python stub 检测、前台 Ctrl+C 处理。本版后还跟了 **40+ Windows-only fix**（taskkill、native PTY、信号差异、路径规范化、file-locking 等等）。

### 减负 wave —— lazy-deps 框架

`tools/lazy_deps.py`（613 行，line 77-173 罗列 25+ 条目）：messaging adapter SDK（Slack/Matrix/Feishu/钉钉）、image-gen SDK（Pixverse/Camofox）、TTS provider、hindsight client、codex app-server、terminal backend（Modal/Daytona/Vercel）……全部**首次使用时**才装。

`pyproject.toml:174-207` `[all]` extras 同步 drop 掉所有 lazy 覆盖的依赖。分级安装 + tier fallback；安装时跑供应链 advisory 扫描。

更小的 wheel、更快的 install、更少的传递依赖漏洞面。

## OAuth Provider 体系大跨进

### xAI Grok SuperGrok OAuth + grok-4.3 升 1M Context

`hermes_cli/auth.py:201` 定义 `xAI Grok OAuth (SuperGrok Subscription)`；`agent/model_metadata.py:217` —— **`grok-4.3` 上下文窗口 1,000,000** tokens（per docs.x.ai）。

SuperGrok 订阅者直接登 xAI 账户即可用，无需 API key/独立计费。带 entitlement error 处理，加 SSH-to-tunnel 文档（远程登 SSH 完成 OAuth 流程）。

### OpenAI-compatible 本地代理（`hermes proxy`）

`hermes_cli/proxy/cli.py:30,78,102`（`cmd_proxy_start / cmd_proxy_status / cmd_proxy_list_providers`）+ `hermes_cli/proxy/server.py` + `hermes_cli/proxy/adapters/xai.py:31` `class XAIGrokAdapter`：

跑 `hermes proxy` 起一个 `http://localhost:port` 端点，**OpenAI API 兼容**，背后是你登的任意 OAuth provider —— Claude Pro、ChatGPT Pro、SuperGrok。Codex CLI / Aider / Cline / Continue / 自定义脚本就能直接用你的订阅，**一个订阅、所有工具**。

### NovitaAI 新 provider

`agent/model_metadata.py` 引用 "novita"、"novita-ai"、"novitaai"、"api.novita.ai" + pricing metadata。开源模型托管（Llama / Qwen / DeepSeek）多一个选项。

### Codex app-server runtime

`agent/codex_runtime.py` + `agent/codex_responses_adapter.py`：drive OpenAI Codex CLI 的可选 runtime，session 复用、wedged-session 自动 retire、OAuth refresh 分类——长 agentic run 不爆掉。

### OpenRouter Pareto Code Router + `min_coding_score`

`cli.py` 解析、`agent/chat_completion_helpers.py` 路由、`tests/providers/test_provider_profiles.py:106` 验证测试：

OpenRouter 的 Pareto router 自动挑"满足质量底线的最便宜模型"。新 `min_coding_score` 配置专门把这个阈值用于 coding 任务 —— **少花顶级模型的钱**。

### Provider 改名：Alibaba Cloud → Qwen Cloud

`agent/model_metadata.py`（"qwen3.6-plus" — DashScope/Alibaba & OpenRouter）+ `agent/agent_runtime_helpers.py` 注释：UI 改名匹配业界叫法，旧 config key 仍生效，无 breaking change。

## 新工具与跨平台能力

### `x_search` —— X / Twitter 一等公民

`tools/x_search_tool.py` + `tools/xai_http.py`：agent 直接搜 X 时间线、找 thread、定位 post。X OAuth 或 API key 鉴权二选一。

### `vision_analyze` —— 像素直传 vision 模型

`tools/vision_tools.py:633` `async def vision_analyze_tool`：当模型支持视觉（GPT-5、Claude、Gemini、Grok-vision）时**直接把原始像素**送进去，不再先转文字描述再 round-trip。

### 统一可插拔 `video_generate`

`tools/video_generation_tool.py`：一个工具、任意视频模型；新视频 model 上线就是一个文件的 plugin，不必 fork。

### `computer_use` cua-driver 后端

`tools/computer_use/cua_backend.py`：默认后端，speaks MCP over stdio to cua-driver；**不再绑死 Anthropic SDK**，任何 vision-capable 模型都能驱动桌面。focus-safe operation + `hermes update` 时自动刷新。安装：`curl` 脚本，`HERMES_CUA_DRIVER_CMD` 覆盖。

### 浏览器：CDP passthrough 180x 加速

`tools/browser_cdp_tool.py:301` `browser_cdp`：所有 `browser_console` evaluation 现在**共享一条到 Chrome 的 WebSocket**，不再每次起新 DevTools session。秒级 → 毫秒级，页面交互如丝般顺滑。

### Web search 后端 + 2 个免费

`tools/web_tools.py`：**Brave Search 免费档**和 **DuckDuckGo (DDGS)** 加入 Tavily / SearXNG / Exa 之列。`_ddgs_package_importable()` 触发 lazy install。无 key 选项。

### LSP 语义诊断（每次写都跑）

`agent/lsp/`（11 modules：`cli.py / protocol.py / client.py / manager.py / eventlog.py / install.py / servers.py / range_shift.py / workspace.py / reporter.py`，总 ~4400 行）：

`write_file` + `patch` 之后跑**真正的 language server**对编辑文件做语义分析 —— 类型错误、未定义符号、缺 import 立即 surface 给 agent。**比 v0.13 的 syntax-only lint 升级一大截。**

### 每轮文件变更 verifier footer

`tools/file_state.py` + `agent/file_safety.py`：每轮如果写/改了文件，**给 agent 一段 footer**总结磁盘上具体变了什么（路径、行数、delta）。Agent 立刻发现"我以为写进去了但实际没保存"的失败，不再自信地报 success。

### Microsoft Teams 端到端

`gateway/platforms/msgraph_webhook.py`（webhook listener）+ `tools/microsoft_graph_auth.py` + `tools/microsoft_graph_client.py` + `plugins/platforms/teams/` + `plugins/teams_pipeline/`（full subscription / meeting / runtime）：

Teams 现在能**读消息也能发回**。Microsoft Graph 整套打通，从任何 Teams 频道、DM、群组都能跟 agent 聊。

## 跨 Session Claude 1h Prompt Cache

`agent/prompt_caching.py:51` `cache_ttl: "5m"`；`agent/model_metadata.py:1268` `_CODEX_OAUTH_CONTEXT_CACHE_TTL = 3600 # 1 hour`：

用 Claude 走 Anthropic / OpenRouter / Nous Portal 时，**system prompt + skills + memory 的 prefix 在 session 之间复用 1 小时**。`/new` 后第一次回复又快又便宜，因为上次的 cache 还热。后台 memory review 也吃 cache。

## API 与插件

### 长 run 审批事件暴露给 HTTP API

不再 silently 卡住——审批请求出现在 stream 上，client 能 prompt 用户并回应。

### Plugin `ctx.llm` + `tool_override`

插件可走 active provider + credentials **直接发 LLM call**，无需手动 client wiring；`tool_override` flag 干净替换 built-in tool。Plugin 作者拿到和 core agent 一样的 routing + auth 基础设施。
